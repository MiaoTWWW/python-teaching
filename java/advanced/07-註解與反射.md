# 07 註解與反射

## 註解（Annotation）

註解是 Java 5 引入的元資料（metadata）機制，為程式碼提供額外資訊，不直接影響程式執行。

### 內建常用註解

```java
@Override  // 告訴編譯器此方法覆寫父類別方法
@Deprecated  // 標記已過時，使用時編譯器產生警告
@SuppressWarnings("unchecked")  // 抑制編譯器警告
@FunctionalInterface  // 函數式介面（只有一個抽象方法）
```

### 自訂註解

```java
import java.lang.annotation.*;

// 定義註解
@Retention(RetentionPolicy.RUNTIME)   // 執行時期仍保留
@Target(ElementType.METHOD)           // 只能用在方法上
public @interface Loggable {
    String value() default "";         // 註解屬性
    Level level() default Level.INFO;  // 列舉屬性
}

public enum Level {
    INFO, WARN, ERROR
}
```

使用自訂註解：

```java
public class MyService {
    @Loggable(value = "執行使用者登入", level = Level.INFO)
    public void login(String username) {
        System.out.println(username + " 登入成功");
    }

    @Loggable(level = Level.ERROR)
    public void dangerousOp() {
        // ...
    }
}
```

### Retention 與 Target

```java
@Retention(RetentionPolicy.SOURCE)    // 原始碼層級（編譯後丟棄，如 @Override）
@Retention(RetentionPolicy.CLASS)     // 編譯後保留（預設，執行時期無法讀取）
@Retention(RetentionPolicy.RUNTIME)   // 執行時期保留（可透過反射讀取）

@Target(ElementType.TYPE)             // 類別、介面
@Target(ElementType.METHOD)           // 方法
@Target(ElementType.FIELD)            // 欄位
@Target(ElementType.PARAMETER)        // 參數
@Target({ElementType.TYPE, ElementType.METHOD})  // 多個目標
```

## 反射（Reflection）

反射讓程式在執行時期動態取得類別資訊、呼叫方法、存取欄位。

### 取得 Class 物件

```java
// 三種方式
Class<?> c1 = String.class;               // 1. .class
Class<?> c2 = "hello".getClass();          // 2. getClass()
Class<?> c3 = Class.forName("java.lang.String");  // 3. Class.forName（最常用）
```

### 動態建立物件與呼叫方法

```java
import java.lang.reflect.*;

public class ReflectionDemo {
    public static void main(String[] args) throws Exception {
        // 動態載入類別
        Class<?> clazz = Class.forName("java.util.ArrayList");

        // 動態建立實例
        List<?> list = (List<?>) clazz.getDeclaredConstructor().newInstance();

        // 動態呼叫 add 方法
        Method addMethod = clazz.getMethod("add", Object.class);
        addMethod.invoke(list, "Hello");
        addMethod.invoke(list, "World");

        // 動態呼叫 size 方法
        Method sizeMethod = clazz.getMethod("size");
        System.out.println(sizeMethod.invoke(list));  // 2

        System.out.println(list);  // [Hello, World]
    }
}
```

### 存取私有欄位

```java
class Person {
    private String name = "Secret";
}

public class ReflectionPrivate {
    public static void main(String[] args) throws Exception {
        Person p = new Person();

        Field field = Person.class.getDeclaredField("name");
        field.setAccessible(true);  // 解除 private 限制

        System.out.println(field.get(p));  // "Secret"
        field.set(p, "Hacked!");
        System.out.println(field.get(p));  // "Hacked!"
    }
}
```

::: warning
反射可以破壞封裝，在框架開發中很常見，但**應用程式程式碼應盡量避免**。
:::

### 讀取執行時期註解

```java
public class AnnotationReader {
    public static void main(String[] args) throws Exception {
        MyService service = new MyService();

        for (Method method : service.getClass().getMethods()) {
            if (method.isAnnotationPresent(Loggable.class)) {
                Loggable log = method.getAnnotation(Loggable.class);
                System.out.println("方法：" + method.getName());
                System.out.println("日誌：" + log.value());
                System.out.println("層級：" + log.level());
            }
        }
    }
}

// 輸出：
// 方法：login
// 日誌：執行使用者登入
// 層級：INFO
// 方法：dangerousOp
// 日誌：
// 層級：ERROR
```

## 反射的實際應用

- **Spring Framework**：透過反射建立 Bean、注入依賴
- **JUnit**：透過反射找到 `@Test` 方法並執行
- **Jackson / Gson**：透過反射將 JSON 轉換為 Java 物件
- **ORM 框架**：透過反射將資料庫紀錄映射到物件

## 重點整理

- 註解為程式碼附加元資料，不直接影響邏輯
- `@Retention(RUNTIME)` 才能透過反射讀取
- 反射在執行時期動態操作類別、方法、欄位
- `setAccessible(true)` 可突破 private 限制
- 反射有效能開銷，避免在高頻路徑使用
- 框架大量使用反射，但應用層應謹慎
