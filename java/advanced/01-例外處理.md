# 01 例外處理

## 什麼是例外？

例外（Exception）是程式執行期間發生的異常狀況，例如除以零、陣列索引超出範圍、讀取不存在的檔案等。

```java
int result = 10 / 0;  // 執行時期拋出 ArithmeticException
```

## try-catch-finally

```java
try {
    int[] arr = {1, 2, 3};
    System.out.println(arr[5]);  // 拋出 ArrayIndexOutOfBoundsException
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("陣列索引超出範圍：" + e.getMessage());
} finally {
    System.out.println("這行一定會執行");  // 資源清理用
}
```

### 多個 catch

```java
try {
    String s = null;
    System.out.println(s.length());   // NullPointerException
    int n = Integer.parseInt("abc");  // NumberFormatException
} catch (NullPointerException e) {
    System.out.println("空指標：" + e.getMessage());
} catch (NumberFormatException e) {
    System.out.println("數字格式錯誤");
} catch (Exception e) {              // 捕撈所有例外（放最後）
    System.out.println("其他錯誤");
}
```

::: tip
例外類別的繼承關係：Exception → RuntimeException（未受檢）/ ...（已受檢），應將最特定的例外寫在前面。
:::

## 受檢例外（Checked）vs 未受檢例外（Unchecked）

| 類別 | 說明 | 範例 | 處理要求 |
|------|------|------|---------|
| **已受檢例外** | 編譯期必須處理 | IOException, SQLException | 必須 try-catch 或 throws |
| **未受檢例外** | 執行期才發生 | NullPointerException, ArithmeticException | 不需強制處理 |

```java
// 已受檢例外：編譯器強制處理
import java.io.*;

public void readFile(String path) {
    try {
        FileReader fr = new FileReader(path);  // 可能拋出 FileNotFoundException
        BufferedReader br = new BufferedReader(fr);
        System.out.println(br.readLine());     // 可能拋出 IOException
        br.close();
    } catch (IOException e) {
        System.out.println("讀檔失敗：" + e.getMessage());
    }
}
```

## throw 與 throws

- 	hrow：手動拋出例外
- 	hrows：在方法簽名宣告可能拋出的例外，呼叫者必須處理

```java
public void setAge(int age) throws IllegalArgumentException {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("年齡不合法：" + age);
    }
}

// 呼叫端
try {
    setAge(200);
} catch (IllegalArgumentException e) {
    System.out.println(e.getMessage());  // 年齡不合法：200
}
```

## try-with-resources（Java 7+）

自動關閉實作 AutoCloseable 的資源（檔案、資料庫連線等）：

```java
// 傳統寫法：需手動 close
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    System.out.println(br.readLine());
} finally {
    if (br != null) br.close();
}

// try-with-resources（推薦）
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    System.out.println(br.readLine());
} catch (IOException e) {
    System.out.println("讀取失敗");
}
// br 會自動關閉，不需寫 finally
```

## 自訂例外

```java
public class InsufficientBalanceException extends Exception {
    public InsufficientBalanceException(String message) {
        super(message);
    }

    public InsufficientBalanceException(String message, double deficit) {
        super(String.format("%s（不足 %.2f 元）", message, deficit));
    }
}

// 使用
public void withdraw(double amount) throws InsufficientBalanceException {
    if (amount > balance) {
        double deficit = amount - balance;
        throw new InsufficientBalanceException("餘額不足", deficit);
    }
    balance -= amount;
}
```

## 重點整理

- try-catch 處理例外，finally 清理資源（幾乎已被 try-with-resources 取代）
- 已受檢例外必須處理，未受檢例外可選
- 	hrow 拋出例外，	hrows 宣告例外
- try-with-resources 自動關閉資源，優先使用
- 自訂例外繼承 Exception（受檢）或 RuntimeException（未受檢）
