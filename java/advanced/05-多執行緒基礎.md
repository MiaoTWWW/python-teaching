# 05 多執行緒基礎

多執行緒讓程式同時執行多個任務，充分利用多核心 CPU。

## 執行緒建立方式

### 方式一：繼承 Thread

```java
class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(getName() + "：" + i);
            try {
                Thread.sleep(500);  // 暫停 500ms
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

// 使用
MyThread t1 = new MyThread();
MyThread t2 = new MyThread();
t1.start();  // 啟動執行緒（注意：不是呼叫 run()）
t2.start();
```

### 方式二：實作 Runnable（推薦）

因為 Java 只能單一繼承，實作 Runnable 更彈性：

```java
class MyTask implements Runnable {
    private final String name;

    public MyTask(String name) {
        this.name = name;
    }

    @Override
    public void run() {
        for (int i = 1; i <= 3; i++) {
            System.out.println(name + "：第 " + i + " 次");
        }
    }
}

// 使用
Thread t1 = new Thread(new MyTask("工作 A"));
Thread t2 = new Thread(new MyTask("工作 B"));
t1.start();
t2.start();
```

### 方式三：Lambda

```java
Thread t = new Thread(() -> {
    System.out.println("Lambda 執行緒執行中");
});
t.start();
```

## 執行緒生命週期

```
新建（New） → 就緒（Runnable） ↔ 執行（Running） → 終止（Terminated）
                    ↑                      ↓
                阻塞（Blocked）← 等待（Waiting）
```

## synchronized 同步

多執行緒同時存取共享資源可能導致資料不一致：

```java
class Counter {
    private int count = 0;

    // synchronized 保證同一時間只有一個執行緒執行此方法
    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

// 測試
Counter counter = new Counter();
Runnable task = () -> {
    for (int i = 0; i < 10000; i++) {
        counter.increment();
    }
};

Thread t1 = new Thread(task);
Thread t2 = new Thread(task);
t1.start();
t2.start();
t1.join();   // 等待 t1 結束
t2.join();   // 等待 t2 結束
System.out.println(counter.getCount());  // 20000（有 synchronized 才正確）
```

### synchronized 區塊

只同步需要保護的關鍵部分，減少效能開銷：

```java
public void process() {
    // 不需同步的操作
    System.out.println("開始處理...");

    // 只有這裡需要同步
    synchronized (this) {
        // 臨界區（critical section）
        count++;
    }
}
```

## volatile

olatile 保證變數的**可見性**（修改立即反映到其他執行緒）：

```java
class RunningFlag {
    private volatile boolean running = true;

    public void stop() {
        running = false;  // 其他執行緒立即看到變更
    }

    public void run() {
        while (running) {
            // 持續執行
        }
    }
}
```

::: tip
olatile 不保證原子性（如 count++ 仍需 synchronized）。
:::

## ExecutorService（執行緒池）

手動管理執行緒容易出錯，建議使用執行緒池：

```java
import java.util.concurrent.*;

public class ThreadPoolExample {
    public static void main(String[] args) {
        // 建立固定大小（4 條）的執行緒池
        ExecutorService executor = Executors.newFixedThreadPool(4);

        // 提交任務
        for (int i = 0; i < 10; i++) {
            int taskId = i;
            executor.submit(() -> {
                System.out.println("任務 " + taskId + " 由 " +
                    Thread.currentThread().getName() + " 執行");
            });
        }

        // 關閉執行緒池
        executor.shutdown();
    }
}
```

## 重點整理

- 建立執行緒：實作 Runnable 優於繼承 Thread
- 執行緒生命週期：新建 → 就緒 → 執行 → 終止
- synchronized 保證原子性與可見性
- olatile 只保證可見性，不保證原子性
- 優先使用 ExecutorService 而非手動 
ew Thread()
- Thread.sleep() 拋出 InterruptedException 需處理
