# 03 泛型（Generic）

泛型讓類別或方法在定義時不指定具體型別，使用時再決定，提升**型別安全**與**程式碼重用**。

## 為什麼需要泛型？

不使用泛型（Java 5 以前）：

```java
List list = new ArrayList();
list.add("Hello");
list.add(123);
String s = (String) list.get(0);  // 需要強制轉型，不型別安全
```

使用泛型：

```java
List<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123);  ← 編譯錯誤！只能放 String
String s = list.get(0);  // 不需轉型，編譯器保證型別安全
```

## 泛型類別

```java
public class Box<T> {     // T：型別參數
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

// 使用
Box<String> stringBox = new Box<>();
stringBox.set("Hello");
String val = stringBox.get();  // 自動推斷型別

Box<Integer> intBox = new Box<>();
intBox.set(100);
```

## 泛型方法

```java
public class Utils {
    // T 只在方法層級有效
    public static <T> T getMiddle(T[] array) {
        return array[array.length / 2];
    }
}

// 使用
String[] names = {"Alice", "Bob", "Charlie"};
String mid = Utils.getMiddle(names);  // "Bob"
```

### 限制型別參數（Bounded Type Parameters）

```java
// T 必須是 Number 或其子類別
public static <T extends Number> double sum(T a, T b) {
    return a.doubleValue() + b.doubleValue();
}

// 使用
double r1 = sum(10, 20);       // Integer（OK）
double r2 = sum(3.14, 2.86);   // Double（OK）
// sum("a", "b");  ← 編譯錯誤！String 不是 Number 的子類別
```

## 萬用字元（Wildcard）

```java
// ? extends：可讀不可寫（Producer）
public void printAll(List<? extends Number> list) {
    for (Number n : list) {
        System.out.println(n);
    }
    // list.add(42);  ← 編譯錯誤！
}

// ? super：可寫不可讀（Consumer）
public void addNumbers(List<? super Integer> list) {
    list.add(10);
    list.add(20);
    Object obj = list.get(0);  // 只能讀成 Object
}
```

### PECS 原則

> **Producer Extends, Consumer Super**
> - 你從集合讀取資料（生產者）→ ? extends
> - 你寫入資料到集合（消費者）→ ? super

## 型別擦除（Type Erasure）

泛型資訊只在**編譯期**有效，執行時期會被擦除：

```java
List<String> strings = new ArrayList<>();
List<Integer> integers = new ArrayList<>();
System.out.println(strings.getClass() == integers.getClass());  // true

// 無法這樣寫：
// if (obj instanceof List<String>)  ← 編譯錯誤！
```

## 重點整理

- 泛型提供編譯期型別安全，消除強制轉型
- T（Type）、E（Element）、K（Key）、V（Value）為常見型別參數命名
- ? extends 用於讀取（生產者），? super 用於寫入（消費者）
- 型別擦除意味著執行時期無法取得泛型型別
- 靜態方法不能使用類別的泛型參數
