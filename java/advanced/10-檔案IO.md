# 10 檔案 I/O 串流

## 什麼是 I/O 串流？

Java 的 I/O（Input/Output）透過**串流（Stream）**來讀寫資料。串流是一連串資料的抽象概念，像是水管一樣，資料從來源流到目的地。

```
資料來源 → 輸入串流 → 程式 → 輸出串流 → 目的地
```

Java 的 I/O 類別主要放在 `java.io` 套件中，分為兩大類：

| 分類 | 說明 | 核心類別 |
|------|------|---------|
| **位元組串流** | 讀寫位元組資料（圖片、影片、任意檔案） | `InputStream`、`OutputStream` |
| **字元串流** | 讀寫文字資料（.txt、.csv） | `Reader`、`Writer` |

## File 類別

`File` 代表檔案或目錄的路徑，並非檔案內容本身：

```java
import java.io.File;

public class FileDemo {
    public static void main(String[] args) {
        File f = new File("C:/data/notes.txt");

        System.out.println("檔名：" + f.getName());       // notes.txt
        System.out.println("路徑：" + f.getPath());        // C:\data\notes.txt
        System.out.println("絕對路徑：" + f.getAbsolutePath());
        System.out.println("存在嗎？" + f.exists());       // false
        System.out.println("是檔案嗎？" + f.isFile());     // false
        System.out.println("是目錄嗎？" + f.isDirectory());// false
        System.out.println("大小：" + f.length());         // 0
    }
}
```

### 目錄操作

File dir = new File("C:/data");

// 列表目錄內容

## 位元組串流（InputStream / OutputStream）

以位元組為單位讀寫，適合處理二進位檔案（圖片、音樂、影片）。

### 讀取檔案－FileInputStream

```java
import java.io.FileInputStream;
import java.io.IOException;

public class ReadBytes {
    public static void main(String[] args) {
        try (FileInputStream fis = new FileInputStream("C:/data/hello.txt")) {
            int data;
            while ((data = fis.read()) != -1) {  // -1 表示讀取完畢
                System.out.print((char) data);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

::: tip
`try-with-resources`（寫法 `try (資源) { }`）會在區塊結束後自動關閉串流，不需要手動呼叫 `close()`。
:::

### 寫入檔案－FileOutputStream

```java
import java.io.FileOutputStream;
import java.io.IOException;

public class WriteBytes {
    public static void main(String[] args) {
        String data = "Hello, Java I/O!";
        try (FileOutputStream fos = new FileOutputStream("C:/data/output.txt")) {
            fos.write(data.getBytes());  // 將字串轉為位元組陣列寫入
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

若想在檔案尾端追加內容，在建構子傳入 `true`：

```java
FileOutputStream fos = new FileOutputStream("C:/data/output.txt", true);
```

### 緩衝區讀寫－BufferedInputStream / BufferedOutputStream

使用緩衝區可大幅提升讀寫效能：

```java
import java.io.*;

public class BufferedCopy {
    public static void main(String[] args) throws IOException {
        try (FileInputStream fis = new FileInputStream("source.jpg");
             BufferedInputStream bis = new BufferedInputStream(fis);
             FileOutputStream fos = new FileOutputStream("copy.jpg");
             BufferedOutputStream bos = new BufferedOutputStream(fos)) {

            byte[] buffer = new byte[8192];  // 8KB 緩衝區
            int bytesRead;
            while ((bytesRead = bis.read(buffer)) != -1) {
                bos.write(buffer, 0, bytesRead);
            }
        }
        System.out.println("複製完成！");
    }
}
```

## 字元串流（Reader / Writer）

以字元為單位讀寫，適合處理純文字檔案。

### 讀取文字檔－FileReader

```java
import java.io.FileReader;
import java.io.IOException;

public class ReadText {
    public static void main(String[] args) {
        try (FileReader fr = new FileReader("C:/data/notes.txt")) {
            int ch;
            while ((ch = fr.read()) != -1) {
                System.out.print((char) ch);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### 寫入文字檔－FileWriter

```java
import java.io.FileWriter;
import java.io.IOException;

public class WriteText {
    public static void main(String[] args) {
        try (FileWriter fw = new FileWriter("C:/data/notes.txt")) {
            fw.write("第一行\n");
            fw.write("第二行\n");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### BufferedReader / BufferedWriter（建議做法）

每次都讀一個字元效率很差，實務上使用緩衝區版本搭配 `readLine()`：

```java
import java.io.*;

public class ReadLineDemo {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(
                new FileReader("C:/data/notes.txt"))) {

            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

```java
import java.io.*;

public class WriteLineDemo {
    public static void main(String[] args) {
        try (BufferedWriter bw = new BufferedWriter(
                new FileWriter("C:/data/notes.txt"))) {

            bw.write("第一行");
            bw.newLine();  // 跨平台換行
            bw.write("第二行");
            bw.newLine();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 物件序列化（ObjectInputStream / ObjectOutputStream）

將 Java 物件轉為位元組串流儲存或傳輸，物件必須實作 `Serializable` 介面：

```java
import java.io.*;

class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class SerializeDemo {
    public static void main(String[] args) throws Exception {
        // 寫入物件
        try (ObjectOutputStream oos = new ObjectOutputStream(
                new FileOutputStream("person.dat"))) {
            oos.writeObject(new Person("小明", 25));
        }

        // 讀取物件
        try (ObjectInputStream ois = new ObjectInputStream(
                new FileInputStream("person.dat"))) {
            Person p = (Person) ois.readObject();
            System.out.println(p.name + "：" + p.age);  // 小明：25
        }
    }
}
```

::: warning
被 `transient` 修飾的欄位不會被序列化。另外，`serialVersionUID` 用於版本控制，建議明確宣告。
:::

## NIO 簡介（java.nio.file）

JDK 7 推出的 NIO（New I/O）提供了更現代化的檔案操作 API：

```java
import java.nio.file.*;
import java.util.List;

public class NIODemo {
    public static void main(String[] args) throws Exception {
        Path path = Paths.get("C:/data/notes.txt");

        // 讀取所有內容（小檔案適用）
        List<String> lines = Files.readAllLines(path);
        for (String line : lines) {
            System.out.println(line);
        }

        // 寫入
        Files.write(path, "新內容".getBytes());

        // 複製
        Files.copy(path, Paths.get("C:/data/backup.txt"),
            StandardCopyOption.REPLACE_EXISTING);
    }
}
```

## 重點整理

- I/O 串流分為**位元組串流**（InputStream/OutputStream）與**字元串流**（Reader/Writer）
- `File` 類別操作檔案與目錄路徑，不處理檔案內容
- `BufferedXxx` 加入緩衝區可大幅提升讀寫效率
- `try-with-resources` 可自動關閉串流資源
- 物件序列化需實作 `Serializable` 介面
- NIO 的 `Files` / `Paths` 提供更簡潔的 API
