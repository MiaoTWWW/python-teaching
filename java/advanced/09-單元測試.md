# 09 單元測試（JUnit）

## 為什麼需要單元測試？

單元測試自動驗證程式中每個獨立單元（通常是方法）是否正確運作：

- **提早發現錯誤** — 修改程式碼後立刻知道是否壞了什麼
- **放心重構** — 測試通過代表行為沒變
- **文件化的規格** — 測試程式碼說明了方法應該怎麼用

## JUnit 5 基礎

JUnit 5 是 Java 最主流的測試框架。在專案中加入：

```xml
<!-- Maven 依賴 -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.11.0</version>
    <scope>test</scope>
</dependency>
```

或 Gradle：

```groovy
testImplementation 'org.junit.jupiter:junit-jupiter:5.11.0'
```

## 第一個測試

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

// 要測試的類別
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int divide(int a, int b) {
        if (b == 0) throw new IllegalArgumentException("不能除以零");
        return a / b;
    }
}

// 測試類別
public class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        int result = calc.add(3, 5);
        assertEquals(8, result);              // 預期 3 + 5 = 8
    }

    @Test
    void testDivideByZero() {
        Calculator calc = new Calculator();
        assertThrows(IllegalArgumentException.class, () -> {
            calc.divide(10, 0);
        });
    }
}
```

## 常用斷言（Assertions）

```java
import static org.junit.jupiter.api.Assertions.*;

@Test
void testAssertions() {
    // 相等
    assertEquals(4, 2 + 2);
    assertEquals(3.14, 3.14159, 0.01);     // 浮點數比較（delta）

    // 布林
    assertTrue(10 > 5);
    assertFalse(10 < 5);

    // null
    assertNull(null);
    assertNotNull("Hello");

    // 陣列
    assertArrayEquals(new int[]{1, 2, 3}, new int[]{1, 2, 3});

    // 同一個物件
    String s = "Hi";
    assertSame(s, s);

    // 例外
    assertThrows(ArithmeticException.class, () -> {
        int x = 10 / 0;
    });

    // 多個斷言全部執行（不因第一個失敗就停）
    assertAll("properties",
        () -> assertEquals(1, 1),
        () -> assertEquals(2, 2),
        () -> assertEquals(3, 3)
    );
}
```

## 測試生命週期

```java
import org.junit.jupiter.api.*;

public class LifecycleTest {

    @BeforeAll
    static void setupAll() {
        System.out.println("在所有測試前執行一次（必須 static）");
    }

    @AfterAll
    static void teardownAll() {
        System.out.println("在所有測試後執行一次（必須 static）");
    }

    @BeforeEach
    void setup() {
        System.out.println("每個測試方法前執行");
    }

    @AfterEach
    void teardown() {
        System.out.println("每個測試方法後執行");
    }

    @Test
    void test1() {
        System.out.println("測試 1");
    }

    @Test
    void test2() {
        System.out.println("測試 2");
    }
}
```

### 執行順序

```
@BeforeAll
  ├─ @BeforeEach
  │  └─ @Test test1()
  ├─ @AfterEach
  ├─ @BeforeEach
  │  └─ @Test test2()
  └─ @AfterEach
@AfterAll
```

## 參數化測試

用不同參數多次執行同一個測試：

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;

public class ParamTest {

    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3, 4, 5})
    void testPositiveNumbers(int num) {
        assertTrue(num > 0);
    }

    @ParameterizedTest
    @CsvSource({
        "1, 1, 2",
        "2, 3, 5",
        "5, 7, 12"
    })
    void testAdd(int a, int b, int expected) {
        Calculator calc = new Calculator();
        assertEquals(expected, calc.add(a, b));
    }

    @ParameterizedTest
    @MethodSource("provideStrings")
    void testUpperCase(String input, String expected) {
        assertEquals(expected, input.toUpperCase());
    }

    static Stream<Arguments> provideStrings() {
        return Stream.of(
            Arguments.of("hello", "HELLO"),
            Arguments.of("java", "JAVA"),
            Arguments.of("test", "TEST")
        );
    }
}
```

## 測試例外

```java
@Test
void testException() {
    // 方式一：assertThrows（推薦）
    Exception e = assertThrows(IllegalArgumentException.class, () -> {
        new Calculator().divide(10, 0);
    });
    assertTrue(e.getMessage().contains("除以零"));

    // 方式二：assertThrowsExactly（只接受完全相同的型別）
    assertThrowsExactly(IllegalArgumentException.class, () -> {
        new Calculator().divide(10, 0);
    });
}
```

## 實務範例：測試一個服務類別

```java
// 要測試的類別
public class UserService {
    private final List<String> users = new ArrayList<>();

    public void register(String username) {
        if (username == null || username.isBlank()) {
            throw new IllegalArgumentException("使用者名稱不能為空");
        }
        if (users.contains(username)) {
            throw new IllegalStateException("使用者已存在");
        }
        users.add(username);
    }

    public List<String> getUsers() {
        return Collections.unmodifiableList(users);
    }

    public boolean exists(String username) {
        return users.contains(username);
    }
}

// 測試
public class UserServiceTest {

    private UserService userService;

    @BeforeEach
    void setup() {
        userService = new UserService();
    }

    @Test
    void testRegisterSuccess() {
        userService.register("Alice");
        assertTrue(userService.exists("Alice"));
        assertEquals(1, userService.getUsers().size());
    }

    @Test
    void testRegisterDuplicate() {
        userService.register("Bob");
        assertThrows(IllegalStateException.class, () -> {
            userService.register("Bob");
        });
    }

    @Test
    void testRegisterBlankName() {
        assertThrows(IllegalArgumentException.class, () -> {
            userService.register(" ");
        });
    }

    @Test
    void testRegisterNull() {
        assertThrows(IllegalArgumentException.class, () -> {
            userService.register(null);
        });
    }
}
```

## 測試覆蓋率概念

測試覆蓋率衡量有多少程式碼被測試執行到：

- **行覆蓋率** — 每一行程式碼是否被執行過
- **分支覆蓋率** — if/else 的每個分支是否都被走過
- **方法覆蓋率** — 每個方法是否被呼叫過

::: tip
追求 100% 覆蓋率不一定實際，關鍵路徑與邊界條件比數字更重要。
:::

## 重點整理

- `@Test` 標記測試方法
- `assertEquals()` `assertTrue()` `assertThrows()` 是最常用的斷言
- `@BeforeEach` / `@AfterEach` 管理測試前後準備與清理
- `@ParameterizedTest` 用多組資料重複測試
- 測試方法名稱應清楚描述**情境與預期結果**
- 測試隔離：每個測試應獨立，不依賴其他測試的結果
