# 04 套件與存取修飾詞

## 套件（Package）

套件是 Java 用來組織類別的命名空間機制：

```java
// 檔案：com/example/school/Student.java
package com.example.school;

public class Student {
    // ...
}
```

### 命名慣例

```
個人專案：com.example.專案名
公司專案：com.公司名.專案名
```

### import 語句

使用其他套件的類別時，需要 import：

```java
import java.util.ArrayList;           // 單一類別
import java.util.*;                    // 該套件所有類別
import static java.lang.Math.PI;       // 靜態匯入（直接使用 PI）

public class Main {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        System.out.println(PI);  // 3.14159...
    }
}
```

::: tip
`java.lang` 套件（`String`、`Math`、`System` 等）會自動匯入，不需寫 `import`。
:::

## 存取修飾詞（Access Modifiers）

| 修飾詞 | 同類別 | 同套件 | 子類別 | 所有類別 |
|--------|--------|--------|--------|---------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| default（不寫） | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

```java
package com.example;

public class AccessExample {
    private int a = 1;       // 僅同類別可存取
    int b = 2;               // 套件私有（同套件可存取）
    protected int c = 3;     // 同套件 + 子類別
    public int d = 4;        // 全部可存取

    private void privateMethod() { }
    public void publicMethod() { }
}
```

## 模組系統（Java 9+）

Java 9 引入模組（module）來強化封裝：

```java
// module-info.java
module com.example.myapp {
    requires java.sql;          // 依賴 java.sql 模組
    exports com.example.api;    // 開放 com.example.api 套件給其他模組使用
    exports com.example.model;
}
```

模組的好處：
- 明確宣告依賴關係
- 控制哪些套件對外公開
- 減少執行時期缺少類別的風險

## 重點整理

- 套件避免類別命名衝突，對應目錄結構
- `import` 匯入其他套件的類別
- 存取修飾詞控制可見度：`private`（最嚴格）→ `public`（最開放）
- 善用 `private` 封裝內部實作細節
- 模組系統（Java 9+）提供更強的封裝邊界
