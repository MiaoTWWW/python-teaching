# 01 類別與物件

## 物件導向簡介

Java 是純物件導向語言，所有程式碼都寫在類別（class）中。

三大特性：
- **封裝** — 隱藏內部實作，只公開必要介面
- **繼承** — 從現有類別衍生新類別
- **多型** — 同一方法在不同物件上有不同行為

## 定義類別

```java
public class Student {
    // 屬性（fields）
    String name;
    int age;
    double score;

    // 建構子（constructor）
    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }

    // 方法（methods）
    public void sayHello() {
        System.out.println("大家好，我是 " + name);
    }
}
```

### 建立物件

```java
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student("Alice", 20, 92.5);
        s1.sayHello();  // 大家好，我是 Alice
    }
}
```

## this 關鍵字

`this` 指向當前物件的參考，常用來區分參數與屬性：

```java
public class Point {
    int x, y;

    public Point(int x, int y) {
        this.x = x;  // this.x 是屬性，x 是參數
        this.y = y;
    }
}
```

## 建構子（Constructor）

### 預設建構子

如果沒有定義任何建構子，Java 會自動提供一個無參數的預設建構子：

```java
public class Dog {
    String name;

    // 編譯器自動加入：
    // public Dog() { }
}
```

### 建構子多載（Overloading）

```java
public class Rectangle {
    int width, height;

    // 無參數建構子
    public Rectangle() {
        this.width = 1;
        this.height = 1;
    }

    // 單參數建構子（正方形）
    public Rectangle(int side) {
        this.width = side;
        this.height = side;
    }

    // 雙參數建構子
    public Rectangle(int width, int height) {
        this.width = width;
        this.height = height;
    }
}
```

### this() 呼叫同類別其他建構子

```java
public class Employee {
    String name;
    String department;

    public Employee(String name) {
        this(name, "一般");  // 呼叫雙參數建構子
    }

    public Employee(String name, String department) {
        this.name = name;
        this.department = department;
    }
}
```

## 方法（Method）

### 回傳值與參數

```java
public class Calculator {
    // 有回傳值
    public int add(int a, int b) {
        return a + b;
    }

    // 無回傳值（void）
    public void printMessage(String msg) {
        System.out.println(msg);
    }
}
```

### 傳值呼叫（Call by Value）

Java 一律採傳值呼叫，方法內修改參數不會影響原始變數：

```java
public class Test {
    public static void change(int x) {
        x = 100;  // 只修改區域變數
    }

    public static void main(String[] args) {
        int a = 10;
        change(a);
        System.out.println(a);  // 10（不受影響）
    }
}
```

::: warning
物件的「參考」也是傳值，所以方法內可修改物件內容，但無法讓參考指向新物件。
:::

## static 關鍵字

`static` 成員屬於類別本身，不屬於個別物件：

```java
public class MathUtils {
    public static final double PI = 3.14159;

    public static int max(int a, int b) {
        return a > b ? a : b;
    }
}

// 使用方式：不需建立物件
double p = MathUtils.PI;
int m = MathUtils.max(10, 20);
```

### static 變數（類別變數）

所有物件共享同一份 static 變數：

```java
public class Counter {
    static int count = 0;  // 所有 Counter 物件共享

    public Counter() {
        count++;
    }
}

// 使用
new Counter();
new Counter();
new Counter();
System.out.println(Counter.count);  // 3
```

### static 方法限制

- static 方法不能直接存取非 static 成員
- static 方法不能使用 `this`

```java
public class Example {
    int x = 10;
    static int y = 20;

    static void staticMethod() {
        // System.out.println(x);  // 編譯錯誤！
        System.out.println(y);      // OK
    }
}
```

## 封裝與 Getter/Setter

封裝（Encapsulation）是 OOP 四大特色之一，目的是將物件的內部資料隱藏起來，僅透過公開方法來存取。

### 為什麼需要封裝？

- 保護資料不被外部任意修改
- 可在設值時加入驗證邏輯
- 可隱藏內部實作細節，降低耦合

### 標準實作模式

將欄位設為 `private`，提供公開的 getter（取值）與 setter（設值）方法：

```java
public class Student {
    private String name;
    private int age;
    private double score;

    // Getter：取得 name
    public String getName() {
        return name;
    }

    // Setter：設定 name
    public void setName(String name) {
        this.name = name;
    }

    // Getter：取得 age
    public int getAge() {
        return age;
    }

    // Setter：設定 age（含驗證）
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年齡不合法：" + age);
            return;
        }
        this.age = age;
    }

    // Getter：取得 score
    public double getScore() {
        return score;
    }

    // Setter：設定 score（自動取 0~100）
    public void setScore(double score) {
        this.score = Math.max(0, Math.min(100, score));
    }
}
```

### 使用封裝的物件

```java
public class TestStudent {
    public static void main(String[] args) {
        Student s = new Student();
        s.setName("小明");
        s.setAge(20);
        s.setScore(95.5);

        System.out.println(s.getName());   // 小明
        System.out.println(s.getAge());     // 20

        // 非法值會被過濾
        s.setAge(200);    // 年齡不合法：200
        s.setScore(150);  // 自動限制為 100
    }
}
```

### Boolean 欄位的 Getter 命名慣例

boolean 型別的 getter 習慣使用 `is` 前綴：

```java
public class Product {
    private boolean available;

    public boolean isAvailable() {  // 不是 getAvailable()
        return available;
    }

    public void setAvailable(boolean available) {
        this.available = available;
    }
}
```

::: tip
現代 IDE（如 IntelliJ IDEA）可一鍵產生 Getter/Setter：右鍵 → Generate → Getter and Setter。
:::

## 重點整理

- 類別是物件的藍圖，物件是類別的實例
- `new` 關鍵字建立物件，呼叫建構子初始化
- `this` 指向當前物件
- 建構子可以多載，`this()` 呼叫同類別其他建構子
- static 成員屬於類別本身，不屬於個別物件
- 封裝透過 `private` 欄位 + 公開 getter/setter 實作，保護資料完整性
