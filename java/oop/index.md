# 物件導向程式設計

> OOP（Object-Oriented Programming）是 Java 的核心哲學。如果基礎篇教你怎麼寫 Java，物件導向篇教你怎麼**設計** Java。

Java 是一套**純物件導向語言**（除基本型別外皆為物件），掌握 OOP 是寫好 Java 的關鍵。

## 章節地圖

| 章節 | 主題 | 你將學會 |
|------|------|---------|
| **01** | 類別與物件 | 類別定義、建構子、this、static、**封裝與 Getter/Setter 設計模式** |
| **02** | 繼承與多型 | extends、super、方法覆寫（Override）與多載（Overload）、向上/向下轉型、final 關鍵字 |
| **03** | 介面與抽象類別 | abstract 抽象類別、interface 介面定義與實作、default 方法、介面 vs 抽象類別選擇時機 |
| **04** | 套件與存取修飾詞 | package 組織類別、public / private / protected 可見度控制、import 與 jar 檔 |
| **05** | 列舉 | enum 基本用法、自訂欄位與方法、switch 搭配 enum、實務狀態機應用 |
| **06** | 巢狀類別 | 靜態巢狀類別、內部類別、區域類別、**匿名類別**的寫法與應用場景 |

## 學習路徑

```
01 類別與物件 ← OOP 的起點
    ↓
02 繼承與多型 ← 讓類別之間產生關係
    ↓
03 介面與抽象類別 ← 定義契約與骨架
    ↓
04 套件與修飾詞 ← 學會組織與保護程式碼
    ↓
05 列舉 + 06 巢狀類別 ← 加分特色，讓程式更精簡
```

## 學完之後你能做到

- 設計出具備封裝性、可維護的類別結構
- 善用繼承與介面建立靈活的軟體架構
- 區分 Override 與 Overload、抽象類別與介面的適用場景
- 使用套件管理大型專案的類別組織
- 用 enum 簡化固定常數集的處理
- 為閱讀主流 Java 框架（Spring、Hibernate）的原始碼做好準備

::: tip
OOP 的概念在進階篇的集合框架、例外處理等章節中會一直用到，不能跳過。
:::
