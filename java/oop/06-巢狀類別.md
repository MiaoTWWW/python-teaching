# 06 巢狀類別（Nested Classes）

巢狀類別是指在類別內部定義的類別，用於將邏輯上相關的類別組織在一起。

## 巢狀類別分類

```
巢狀類別
  ├─ 靜態巢狀類別（static nested class）
  └─ 內部類別（inner class）
       ├─ 成員內部類別（member inner class）
       ├─ 區域內部類別（local inner class）
       └─ 匿名內部類別（anonymous class）
```

## 靜態巢狀類別（Static Nested Class）

使用 `static` 修飾，行為類似一般的頂層類別，但被巢狀在另一個類別內部：

```java
public class Computer {
    private String brand;

    public Computer(String brand) {
        this.brand = brand;
    }

    public void start() {
        System.out.println(brand + " 開機中");
    }

    // 靜態巢狀類別
    public static class Motherboard {
        private String model;

        public Motherboard(String model) {
            this.model = model;
        }

        public void showInfo() {
            System.out.println("主機板型號：" + model);
            // System.out.println(brand);  // 編譯錯誤！不能存取外部類別的實例成員
        }
    }
}

// 使用
Computer.Motherboard mb = new Computer.Motherboard("Z790");
mb.showInfo();  // 主機板型號：Z790
```

::: tip
靜態巢狀類別不持有外部類別的參考，不能存取外部類別的實例成員（但可存取 static 成員）。
:::

## 成員內部類別（Member Inner Class）

沒有 `static` 修飾，可以直接存取外部類別的所有成員（包括 private）：

```java
public class Outer {
    private String message = "Hello from Outer";

    public class Inner {
        public void print() {
            System.out.println(message);  // 可直接存取外部 private 成員
        }

        public Outer getOuter() {
            return Outer.this;  // 取得外部類別參考
        }
    }

    public Inner createInner() {
        return new Inner();
    }
}

// 使用
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();    // 需要透過外部物件建立
inner.print();  // Hello from Outer
```

### 內部類別編譯結果

內部類別編譯後會產生 `Outer$Inner.class` 檔案。內部類別會持有一個指向外部類別物件的隱藏參考。

## 區域內部類別（Local Inner Class）

定義在方法或區塊內的類別，只在該方法內可見：

```java
public class LocalOuter {
    public void sayHello(String name) {
        // 區域內部類別（在方法內定義）
        class Greeting {
            void greet() {
                System.out.println("Hello, " + name);
            }
        }

        Greeting g = new Greeting();
        g.greet();
    }
}

// 使用
LocalOuter lo = new LocalOuter();
lo.sayHello("Alice");  // Hello, Alice
```

區域內部類別的限制：
- 只能在定義它的方法/區塊內使用
- 存取區域變數時，該變數必須是 `final` 或 effectively final

## 匿名內部類別（Anonymous Class）

沒有名稱的內部類別，適合只用一次的情境：

```java
// 實作介面
Runnable task = new Runnable() {
    @Override
    public void run() {
        System.out.println("匿名類別執行中");
    }
};

// 繼承類別並覆寫方法
List<String> list = new ArrayList<>() {
    @Override
    public boolean add(String s) {
        System.out.println("新增：" + s);
        return super.add(s);
    }
};
```

### 匿名類別 vs Lambda

```java
// 匿名內部類別（傳統）
button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("按鈕被點擊");
    }
});

// Lambda（現代，僅限函數式介面）
button.addActionListener(e -> System.out.println("按鈕被點擊"));
```

::: tip
如果介面只有一個抽象方法（函數式介面），優先使用 Lambda 而非匿名類別。
:::

## 巢狀類別的使用場景

| 類型 | 適用場景 |
|------|----------|
| **靜態巢狀類別** | 輔助類別與外部類別邏輯相關，但不需要存取外部實例（如 `Map.Entry`） |
| **成員內部類別** | 需要存取外部類別實例的輔助類別（如 `ArrayList.Itr`） |
| **區域內部類別** | 只在一個方法內需要的類別 |
| **匿名內部類別** | 只用一次的簡單實作（Lambda 出現後大部分已被取代） |

## 重點整理

- 靜態巢狀類別不持有外部類別參考，行為最接近獨立類別
- 成員內部類別可以存取外部類別的所有成員（包括 private）
- 區域內部類別只能在定義它的方法內使用
- 匿名內部類別適合一次性實作，函數式介面應優先使用 Lambda
- 內部類別編譯為 `Outer$Inner.class` 檔案
