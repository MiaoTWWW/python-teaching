# 03 介面與抽象類別

## 抽象類別（Abstract Class）

用 bstract 宣告的類別，可包含抽象方法（只有宣告，沒有實作）：

```java
public abstract class Shape {
    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    // 抽象方法：子類別必須實作
    public abstract double getArea();

    // 一般方法：可直接使用
    public String getColor() {
        return color;
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

public class Rectangle extends Shape {
    private double width, height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double getArea() {
        return width * height;
    }
}
```

使用：

```java
// Shape s = new Shape("red");  ← 錯誤！抽象類別不能直接 new
Shape c = new Circle("red", 5);
Shape r = new Rectangle("blue", 4, 6);
System.out.println(c.getArea());  // 78.5398...
```

## 介面（Interface）

介面定義類別「必須做什麼」，但不管「怎麼做」。Java 8+ 介面可包含 default 與 static 方法：

```java
// 定義介面
public interface Flyable {
    void fly();  // 抽象方法（不需寫 abstract）
}

public interface Swimmable {
    void swim();
}

// 實作多個介面（Java 的多重繼承方案）
public class Duck implements Flyable, Swimmable {
    private String name;

    public Duck(String name) {
        this.name = name;
    }

    @Override
    public void fly() {
        System.out.println(name + " 正在飛翔");
    }

    @Override
    public void swim() {
        System.out.println(name + " 正在游泳");
    }
}
```

### default 方法（Java 8+）

在介面中提供預設實作，實作類別可選擇是否覆寫：

```java
public interface Vehicle {
    void start();

    default void honk() {
        System.out.println("喇叭聲：嗶嗶！");
    }
}

public class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("汽車發動");
    }
    // honk() 直接用預設實作，不需實作
}
```

### 函數式介面（@FunctionalInterface）

只有**一個**抽象方法的介面，可用 Lambda 表達式實作：

```java
@FunctionalInterface
public interface Calculator {
    int calculate(int a, int b);
}

// 使用 Lambda
Calculator add = (a, b) -> a + b;
Calculator multiply = (a, b) -> a * b;
System.out.println(add.calculate(3, 5));     // 8
System.out.println(multiply.calculate(3, 5)); // 15
```

## 介面 vs 抽象類別

| 比較 | 抽象類別 | 介面 |
|------|---------|------|
| 關鍵字 | bstract class | interface |
| 繼承 | 單一繼承（extends） | 多重實作（implements） |
| 建構子 | ✅ 可有 | ❌ 不行 |
| 欄位 | 任意型別 | 只能是 public static final 常數 |
| 方法 | 抽象 + 一般方法 | 抽象 + default + static |
| private 方法 | ✅ | ✅（Java 9+） |

## Record 類別（Java 16+）

不可變資料載體，自動產生建構子、getter、equals()、hashCode()、	oString()：

```java
public record Point(int x, int y) { }

// 等同於手動寫一長串
Point p = new Point(3, 4);
System.out.println(p.x());     // 3（自動產生 getter）
System.out.println(p);         // Point[x=3, y=4]
```

## 重點整理

- 抽象類別用 extends 繼承，介面用 implements 實作
- Java 可實作多個介面來彌補單一繼承的限制
- @FunctionalInterface 可用 Lambda 簡化實作
- ecord 自動生成樣板程式碼，適合單純的資料載體
