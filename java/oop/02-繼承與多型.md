# 02 繼承與多型

## 繼承（Inheritance）

使用 `extends` 讓子類別繼承父類別的欄位與方法：

```java
// 父類別
public class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " 正在吃東西");
    }

    public void sleep() {
        System.out.println(name + " 正在睡覺");
    }
}

// 子類別
public class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 呼叫父類別建構子
    }

    public void bark() {
        System.out.println(name + "：汪汪！");
    }
}

public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    public void meow() {
        System.out.println(name + "：喵喵～");
    }
}
```

使用：

```java
Dog dog = new Dog("Buddy");
dog.eat();    // Buddy 正在吃東西（繼承自 Animal）
dog.bark();   // Buddy：汪汪！（自己的方法）
```

::: warning
- 子類別建構子第一行必須呼叫 `super()`（若不寫，編譯器自動加無參數版本）
- Java 只支援**單一繼承**（一個類別只能有一個直接父類別）
:::

## super 關鍵字

`super` 用來存取父類別的成員：

```java
public class Parent {
    public String name = "Parent";

    public void show() {
        System.out.println("Parent show");
    }
}

public class Child extends Parent {
    public String name = "Child";

    public void show() {
        System.out.println(super.name);  // Parent
        super.show();                    // Parent show
        System.out.println(this.name);   // Child
    }
}
```

## 方法覆寫（Override）

子類別重新定義父類別的方法，使用 `@Override` 讓編譯器檢查簽名是否一致：

```java
public class Dog extends Animal {
    @Override
    public void eat() {
        System.out.println(name + " 正在大口吃肉");
    }
}
```

覆寫規則：
- 方法簽名（名稱 + 參數）必須完全相同
- 回傳型別可以是原型的子型別（共變回傳）
- 存取修飾詞不能更嚴格（可更寬鬆）
- 父類別的 `final` 方法不能被覆寫

## 多型（Polymorphism）

父類別參考變數可以指向子類別物件，執行時動態決定呼叫哪個方法：

```java
Animal a1 = new Dog("Buddy");
Animal a2 = new Cat("Kitty");

a1.eat();  // Buddy 正在大口吃肉（執行 Dog 的版本）
a2.eat();  // Kitty 正在吃東西（執行 Cat 的版本，因為沒覆寫）

// a1.bark();  ← 編譯錯誤！Animal 型別沒有 bark 方法
```

### instanceof 與向下轉型

檢查物件實際型別，再安全地轉型：

```java
if (a1 instanceof Dog) {
    Dog dog = (Dog) a1;  // 向下轉型（downcasting）
    dog.bark();
}

// Java 16+ 的 pattern matching
if (a1 instanceof Dog d) {
    d.bark();  // 不需另外轉型
}
```

## Object 類別

Java 中所有類別都隱式繼承 `java.lang.Object`：

| 方法 | 用途 |
|------|------|
| `toString()` | 回傳物件的字串表示 |
| `equals(Object)` | 比較兩個物件是否相等 |
| `hashCode()` | 回傳物件的雜湊碼 |
| `getClass()` | 回傳執行時期型別 |
| `clone()` | 建立物件的淺拷貝 |

### 覆寫 toString()、equals()、hashCode()

```java
public class Point {
    int x, y;

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    @Override
    public String toString() {
        return "(" + x + ", " + y + ")";
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj instanceof Point p) {
            return this.x == p.x && this.y == p.y;
        }
        return false;
    }

    @Override
    public int hashCode() {
        return 31 * x + y;
    }
}
```

使用：

```java
Point p1 = new Point(3, 4);
Point p2 = new Point(3, 4);
System.out.println(p1);              // (3, 4)
System.out.println(p1.equals(p2));   // true
```

::: tip
覆寫 `equals()` 時一定要同時覆寫 `hashCode()`，否則 HashMap、HashSet 等集合會出錯。
:::

## final 關鍵字

- `final class`：不可被繼承（如 `String`、`Math`）
- `final method`：不可被覆寫
- `final variable`：不可被重新賦值

```java
final class MathUtils {
    // 此類別不能有子類別
}

class Parent {
    final void cannotOverride() { }
}
```

## 重點整理

- `extends` 實作繼承，`super()` 呼叫父類別建構子
- `@Override` 覆寫父類別方法，簽名必須一致
- 多型：父類別型別參考指向子類別物件，執行期動態綁定
- `instanceof` 檢查實際型別，再進行向下轉型
- Java 只有單一繼承，但可用介面實現多重繼承效果
- 覆寫 `equals()` 時必須同時覆寫 `hashCode()`
