# 01 - 環境安裝與設定

> 🎯 **學習目標**：在本章中，你將學習如何安裝 Python、設定開發環境，並執行你的第一行程式碼。

## 什麼是 Python？

Python 是一種**高階、直譯式**的程式語言，由 Guido van Rossum 於 1991 年創造。它以簡潔易讀的語法聞名，被廣泛應用於：

- 🌐 網路開發
- 📊 資料科學與機器學習
- 🤖 自動化腳本
- 🎮 遊戲開發
- 🔬 科學計算

## 安裝 Python

### Windows 系統

1. 前往 [Python 官方網站](https://www.python.org/downloads/)
2. 點擊下載最新版本的 Python（建議 3.11 以上）
3. 執行安裝程式，**務必勾選** ✅ `Add Python to PATH`
4. 點擊 `Install Now` 完成安裝

![Python 安裝畫面](https://docs.python.org/zh-tw/3/_images/win_installer.png)

### 驗證安裝

開啟終端機（命令提示字元），輸入以下指令：

```bash
python --version
```

如果看到類似以下的輸出，表示安裝成功：

```
Python 3.12.0
```

## 開發環境選擇

你可以選擇以下任一方式來撰寫 Python 程式：

### 4️⃣ Google Colab（雲端開發）

[Google Colab](https://colab.research.google.com/) 是 Google 提供的**雲端 Jupyter Notebook 服務**，無需安裝任何軟體，直接在瀏覽器中撰寫與執行 Python 程式。

**優點：**
- ✅ **零安裝** — 開啟瀏覽器即可使用
- ✅ **免費 GPU** — 可免費使用 GPU 加速機器學習運算
- ✅ **雲端儲存** — 檔案自動儲存在 Google Drive
- ✅ **協作共享** — 像 Google Docs 一樣分享筆記本

4. 按 `Shift+Enter` 執行程式碼

**匯入本課程的 .ipynb 檔案：**

## 虛擬環境

虛擬環境可以讓每個專案擁有獨立的套件環境，避免套件衝突。

### 使用 venv 建立虛擬環境

```bash
# 建立虛擬環境
python -m venv myenv

# 啟動虛擬環境（Windows）
myenv\Scripts\activate

# 啟動虛擬環境（macOS/Linux）
source myenv/bin/activate

# 離開虛擬環境
deactivate
```

### 使用 pip 安裝套件

```bash
# 安裝套件
pip install requests

# 安裝特定版本
pip install requests==2.31.0

# 列出已安裝套件
pip list

# 匯出套件列表
pip freeze > requirements.txt

# 從套件列表安裝
pip install -r requirements.txt
```

## 程式碼可讀性與 PEP 8

Python 的一大特色是**強調程式碼的可讀性**。PEP 8 是 Python 的官方程式碼風格指南，遵循它可以讓你的程式碼更容易被他人（以及未來的自己）閱讀。

### 命名慣例

```python
# ✅ 好的命名 — 一看就懂
user_name = "Alice"
total_score = 95
is_active = True
MAX_RETRIES = 3          # 常數用全大寫

# ❌ 不好的命名 — 難以理解
u = "Alice"
ts = 95
flag = True
x = 3
```

### 縮排與空白

```python
# ✅ 4 個空格縮排（不要使用 Tab）
if score >= 60:
    print("及格！")         # 縮排 4 個空格
    if score >= 90:
        print("優等！")     # 巢狀再縮 4 個空格

# ✅ 運算子兩邊各留一個空白
result = (a + b) * c / d
# ❌ result = (a+b)*c/d     ← 擠在一起難閱讀
```

### 每行程式碼長度

```python
# ✅ 建議每行不超過 79 個字元
# 太長時可以用括號或反斜線換行
total = (item1 + item2 + item3 + item4 + item5
         + item6 + item7 + item8)
```

### Import 順序

```python
# ✅ 標準順序：標準函式庫 → 第三方套件 → 自訂模組
import os
import sys

import requests
import pandas as pd

from mymodule import my_function
```

> 💡 **養成習慣**：寫程式時隨時注意可讀性，好的命名 + 一致的風格 = 好維護的程式碼。

## 善用 AI 輔助學習

寫程式遇到問題時，學會**問對問題**是重要的技能。以下是一些技巧：

### 如何問 AI 問題

```python
# ❌ 模糊的問題
# 「我的程式壞了，幫我看」

# ✅ 具體的問題（附上錯誤訊息）
# 「Python 報錯 TypeError: unsupported operand type(s)
#  for +: 'int' and 'str'，我的程式碼如下：
#
#  age = input("請輸入年齡：")
#  print("明年你" + age + 1 + "歲")  # 這裡報錯
#
#  請問為什麼？該如何修正？」

# ✅ 更好的問法（先說明想達成的目標）
# 「我想讓使用者輸入年齡，然後輸出明年幾歲。
#  我寫了 input() 但合併字串和數字時出錯，
#  該怎麼正確轉換型別？」
```

### 教中學（Learning by Teaching）

教學相長，**把學到的概念解釋給別人聽**是驗證自己是否真正理解的最好方法：

```python
# 想像你要教同學「變數」這個概念，試著用你自己的話解釋：
#
# 「變數就像一個貼了標籤的盒子，
#  標籤是變數名稱，盒子裡面裝的是資料。
#  你可以把舊的資料拿出來，放新的進去，
#  但標籤（變數名稱）不會變。」
```

**建議練習：**
1. 學完一個主題後，試著用一段話解釋給 AI 聽
2. 請 AI 扮演完全不懂的初學者，問你問題
3. 用現實生活中的比喻來解釋程式概念
4. 如果解釋不出來，代表你還沒完全理解 — 這就是需要回頭複習的信號！

> 💡 **課堂活動**：老師會不定期安排「教中學」環節，請同學互相教學，並使用 AI 作為你的「驗證夥伴」來確認你的解釋是否正確。
