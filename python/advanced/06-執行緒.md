## 1. 行程 vs 執行緒

| 面向 | 行程 (Process) | 執行緒 (Thread) |
|------|----------------|------------------|
| 記憶體 | 獨立位址空間，彼此隔離 | 共用相同記憶體空間 |
| 通訊 | IPC (pipe, socket, shared memory) | 直接讀寫共用變數 |
| 建立成本 | 高 (需複製資源) | 低 (輕量級) |
| 切換開銷 | 較大 (核心態切換) | 較小 (同一行程內) |
| 適合場景 | CPU 密集型、需隔離 | I/O 密集型、低延遲協作 |

```python
# 行程：各自獨立的記憶體空間
from multiprocessing import Process

# 執行緒：共用記憶體空間
from threading import Thread
```

## 2. threading.Thread — 基本用法

### 2.1 使用目標函數 (target function)

```python
import threading
import time

def worker(name: str, delay: float):
    print(f"[{name}] 開始工作")
    time.sleep(delay)
    print(f"[{name}] 工作完成 (延遲 {delay} 秒)")

# 建立執行緒
t1 = threading.Thread(target=worker, args=("A", 1.0))
t2 = threading.Thread(target=worker, args=("B", 0.5))

# 啟動執行緒
t1.start()
t2.start()

# 等待執行緒結束
t1.join()
t2.join()

print("所有執行緒已完成")
```

```
[A] 開始工作
[B] 開始工作
[B] 工作完成 (延遲 0.5 秒)
[A] 工作完成 (延遲 1.0 秒)
所有執行緒已完成
```

### 2.2 start()、join() 與生命週期

- **`start()`** — 讓執行緒開始執行，呼叫後立即返回
- **`join(timeout=None)`** — 阻塞直到執行緒結束，可設定超時秒數
- **`is_alive()`** — 檢查執行緒是否仍在執行
- **`name`** — 執行緒名稱，可在建立時指定

```python
t = threading.Thread(target=worker, args=("X", 2), name="Worker-X")
print(f"啟動前: {t.is_alive()}")
t.start()
print(f"啟動後: {t.is_alive()}")
t.join(timeout=1.0)       # 最多等 1 秒
print(f"join 後: {t.is_alive()}")
```

```
啟動前: False
啟動後: True
join 後: True   # 因為 timeout=1 < sleep(2)
```

## 3. 函數式 vs 類別式執行緒

### 3.1 函數式 (Function-based)

```python
import threading

def print_numbers():
    for i in range(5):
        print(f"數字: {i}")

threading.Thread(target=print_numbers).start()
```

### 3.2 類別式 (Class-based)

繼承 `Thread` 並覆寫 `run()`：

```python
import threading
import time

class TimerThread(threading.Thread):
    def __init__(self, name: str, duration: float):
        super().__init__(name=name)
        self.duration = duration

    def run(self):
        start = time.time()
        while time.time() - start < self.duration:
            remaining = self.duration - (time.time() - start)
            print(f"[{self.name}] 剩餘 {remaining:.1f} 秒")
            time.sleep(0.5)
        print(f"[{self.name}] 倒數完成")

t = TimerThread("TimerA", 2.0)
t.start()
t.join()
```

```
[TimerA] 剩餘 2.0 秒
[TimerA] 剩餘 1.5 秒
[TimerA] 剩餘 1.0 秒
[TimerA] 剩餘 0.5 秒
[TimerA] 倒數完成
```

## 4. Daemon 與非 Daemon 執行緒

| 特性 | Daemon | Non-Daemon |
|------|--------|------------|
| 主程式結束時 | 立即強制終止 | 等待執行緒完成 |
| 設定的時機 | `start()` 之前 | — |
| 典型用途 | 背景監控、GC、心跳 | 重要的工作任務 |

```python
import threading
import time

def background_task():
    while True:
        print("背景執行緒工作中...")
        time.sleep(1)

def important_task():
    time.sleep(3)
    print("重要任務已完成")

daemon_t = threading.Thread(target=background_task, daemon=True)   # daemon=True
normal_t = threading.Thread(target=important_task)

daemon_t.start()
normal_t.start()

# 主程式只等 normal_t，不等 daemon_t
normal_t.join()
print("主程式結束 — daemon 執行緒將被強制終止")
```

```
背景執行緒工作中...
背景執行緒工作中...
背景執行緒工作中...
重要任務已完成
主程式結束 — daemon 執行緒將被強制終止
```

> **注意**：Daemon 執行緒不會被等待，主程式結束時會被突然終止，因此不適合用於需要清理資源的任務。

## 5. 競爭危害 (Race Condition)

當多個執行緒同時讀寫「共用變數」且至少一個進行寫入，就可能發生競爭危害。

```python
import threading

counter = 0  # 共用變數
ITERATIONS = 100_000

def increment():
    global counter
    for _ in range(ITERATIONS):
        # 這行實際上不是原子操作：
        # 1. 讀取 counter
        # 2. 計算 counter + 1
        # 3. 寫回 counter
        counter += 1

t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)
t1.start()
t2.start()
t1.join()
t2.join()

expected = ITERATIONS * 2
print(f"預期值: {expected}")
print(f"實際值: {counter}")
print(f"差距:   {expected - counter}  ← 這就是競爭危害的結果")
```

```
預期值: 200000
實際值: 182347  ← 每次執行結果可能不同
差距:   17653
```

> **原因**：`counter += 1` 在 Python 中對應三條位元組碼 (LOAD + ADD + STORE)，執行緒可能在讀取後、寫入前被切換，造成更新遺失。

## 6. 同步機制

### 6.1 Lock (Mutex)

`Lock` 保證同一時間只有一個執行緒可以進入臨界區 (critical section)。

```python
import threading

counter = 0
ITERATIONS = 100_000
lock = threading.Lock()

def safe_increment():
    global counter
    for _ in range(ITERATIONS):
        # 方法一：with 陳述式（建議）
        with lock:
            counter += 1

        # 方法二：手動 acquire / release（等價）
        # lock.acquire()
        # counter += 1
        # lock.release()

t1 = threading.Thread(target=safe_increment)
t2 = threading.Thread(target=safe_increment)
t1.start()
t2.start()
t1.join()
t2.join()

print(f"預期值: {ITERATIONS * 2}, 實際值: {counter}")
```

```
預期值: 200000, 實際值: 200000
```

### 6.2 RLock (可重入鎖)

`RLock` 允許**同一個執行緒**多次 `acquire()` 而不死結，適用於遞迴或巢狀鎖定。

```python
lock = threading.Lock()
rlock = threading.RLock()

def recursive_lock_demo(n: int):
    # 使用 Lock 時這裡會死結
    rlock.acquire()
    if n > 0:
        print(f"遞迴層級: {n}")
        recursive_lock_demo(n - 1)
    rlock.release()

t = threading.Thread(target=recursive_lock_demo, args=(3,))
t.start()
t.join()
```

```
遞迴層級: 3
遞迴層級: 2
遞迴層級: 1
```

### 6.3 Semaphore (信號量)

`Semaphore` 控制同時存取資源的執行緒數量。

```python
import threading
import time

pool = threading.Semaphore(3)  # 最多 3 個同時連線

def db_query(query_id: int):
    with pool:
        print(f"[查詢 {query_id}] 取得連線")
        time.sleep(1.0)
        print(f"[查詢 {query_id}] 釋放連線")

threads = [threading.Thread(target=db_query, args=(i,)) for i in range(6)]
for t in threads:
    t.start()
for t in threads:
    t.join()
```

```
[查詢 0] 取得連線
[查詢 1] 取得連線
[查詢 2] 取得連線
[查詢 0] 釋放連線
[查詢 3] 取得連線
[查詢 1] 釋放連線
[查詢 4] 取得連線
[查詢 2] 釋放連線
[查詢 5] 取得連線
[查詢 3] 釋放連線
[查詢 4] 釋放連線
[查詢 5] 釋放連線
```

## 7. Event (事件信號)

`Event` 讓一個執行緒通知其他執行緒某個條件已成立。

```python
import threading
import time
import random

ready_event = threading.Event()
data_ready = threading.Event()
data = [None]

def producer():
    time.sleep(random.uniform(0.5, 1.5))
    data[0] = f"重要資料 #{random.randint(100, 999)}"
    print(f"[生產者] 資料已產生: {data[0]}")
    data_ready.set()  # 通知消費者

def consumer():
    print("[消費者] 等待資料...")
    data_ready.wait()  # 阻塞直到被 set
    print(f"[消費者] 收到資料: {data[0]}")
    data_ready.clear()

t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer)
t2.start()
t1.start()
t1.join()
t2.join()
```

```
[消費者] 等待資料...
[生產者] 資料已產生: 資料 #512
[消費者] 收到資料: 資料 #512
```

## 8. Queue — 生產者-消費者模式

`queue.Queue` 是**執行緒安全**的 FIFO 佇列，內部已使用 Lock，無需額外同步。

```python
import threading
import queue
import time
import random

q = queue.Queue(maxsize=5)

def producer(name: str, count: int):
    for i in range(count):
        item = f"{name}-商品#{i}"
        q.put(item)
        print(f"[生產者 {name}] 放入: {item}  (佇列大小: {q.qsize()})")
        time.sleep(random.uniform(0.1, 0.3))
    q.put(None)  # 結束信號

def consumer(name: str):
    while True:
        item = q.get()
        if item is None:
            q.task_done()
            break
        print(f"[消費者 {name}] 取出: {item}")
        time.sleep(random.uniform(0.2, 0.5))
        q.task_done()

prod = threading.Thread(target=producer, args=("工廠A", 5))
cons = threading.Thread(target=consumer, args=("小明",))

prod.start()
cons.start()
prod.join()
q.join()      # 等待所有 task_done()
print("所有商品已處理完畢")
```

```
[生產者 工廠A] 放入: 工廠A-商品#0  (佇列大小: 1)
[生產者 工廠A] 放入: 工廠A-商品#1  (佇列大小: 2)
[消費者 小明] 取出: 工廠A-商品#0
[生產者 工廠A] 放入: 工廠A-商品#2  (佇列大小: 2)
[消費者 小明] 取出: 工廠A-商品#1
[生產者 工廠A] 放入: 工廠A-商品#3  (佇列大小: 2)
[生產者 工廠A] 放入: 工廠A-商品#4  (佇列大小: 3)
[消費者 小明] 取出: 工廠A-商品#2
[消費者 小明] 取出: 工廠A-商品#3
[消費者 小明] 取出: 工廠A-商品#4
所有商品已處理完畢
```

## 9. GIL (Global Interpreter Lock)

### 9.1 GIL 是什麼？

GIL 是 CPython 的**全域直譯器鎖**，保證同一時間只有**一個執行緒**可以執行 Python 位元組碼。

### 9.2 GIL 的影響

```python
import threading
import time

def cpu_intensive(n: int) -> int:
    """純 CPU 計算 — 受 GIL 限制"""
    result = 0
    for i in range(n):
        result += i ** 2
    return result

def io_simulation(duration: float):
    """模擬 I/O 等待 — 不受 GIL 影響"""
    time.sleep(duration)

# CPU 密集型測試
start = time.time()
threads = [threading.Thread(target=cpu_intensive, args=(10_000_000,))
           for _ in range(4)]
for t in threads: t.start()
for t in threads: t.join()
cpu_time = time.time() - start

# I/O 密集型測試
start = time.time()
threads = [threading.Thread(target=io_simulation, args=(1.0,))
           for _ in range(4)]
for t in threads: t.start()
for t in threads: t.join()
io_time = time.time() - start

print(f"CPU 密集型 (4 執行緒): {cpu_time:.2f} 秒  ← 不會比單執行緒快")
print(f"I/O  密集型 (4 執行緒): {io_time:.2f} 秒  ← 接近 1 秒")
```

```
CPU 密集型 (4 執行緒): 8.34 秒  ← 不會比單執行緒快
I/O  密集型 (4 執行緒): 1.01 秒  ← 接近 1 秒
```

### 9.3 建議策略

| 任務類型 | 適合工具 | 原因 |
|----------|----------|------|
| **I/O 密集型** (網路請求、檔案讀寫、DB 查詢) | `threading` / `asyncio` | 等待期間釋放 GIL |
| **CPU 密集型** (影像處理、數值計算) | `multiprocessing` | 每個行程有獨立 GIL |
| **混合型** | `ThreadPoolExecutor` + `ProcessPoolExecutor` | 依子任務類型分配 |

## 10. concurrent.futures.ThreadPoolExecutor

更簡潔的執行緒池管理：

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import time

def square(n: int) -> tuple[int, int]:
    time.sleep(0.5)
    return n, n * n

with ThreadPoolExecutor(max_workers=4) as executor:
    # submit 傳回 Future 物件
    futures = [executor.submit(square, i) for i in range(8)]

    for future in as_completed(futures):
        n, n2 = future.result()
        print(f"{n}² = {n2}")
```

```
0² = 0
2² = 4
1² = 1
3² = 9
5² = 25
6² = 36
7² = 49
4² = 16
```

使用 `map()` 簡化：

```python
with ThreadPoolExecutor(max_workers=4) as executor:
    results = executor.map(square, range(8))
    for n, n2 in results:
        print(f"{n}² = {n2}")
```

```
0² = 0
1² = 1
2² = 4
3² = 9
4² = 16
5² = 25
6² = 36
7² = 49
```

> **`map()` vs `submit()`**：`map()` 保持輸入順序；`submit()` + `as_completed()` 按完成順序取得結果。

## 11. 實戰：並發下載 URL

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import urllib.request
import time

URLS = [
    "https://httpbin.org/delay/1",
    "https://httpbin.org/delay/2",
    "https://httpbin.org/delay/3",
    "https://httpbin.org/delay/1",
    "https://httpbin.org/delay/2",
]

def fetch_url(url: str, timeout: int = 10) -> tuple[str, int, float]:
    """下載 URL，回傳 (url, 狀態碼, 耗時秒數)"""
    start = time.time()
    with urllib.request.urlopen(url, timeout=timeout) as resp:
        status = resp.status
    elapsed = time.time() - start
    return url, status, elapsed

def download_all(urls: list[str], max_workers: int = 5):
    print(f"使用 {max_workers} 個執行緒下載 {len(urls)} 個 URL")
    start = time.time()
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        future_to_url = {executor.submit(fetch_url, url): url for url in urls}
        for future in as_completed(future_to_url):
            url, status, elapsed = future.result()
            print(f"[✓] {url} -> {status} ({elapsed:.2f}s)")
    total = time.time() - start
    print(f"總耗時: {total:.2f}s")

download_all(URLS, max_workers=5)
```

```
使用 5 個執行緒下載 5 個 URL
[✓] https://httpbin.org/delay/1 -> 200 (1.23s)
[✓] https://httpbin.org/delay/3 -> 200 (3.12s)
[✓] https://httpbin.org/delay/1 -> 200 (1.18s)
[✓] https://httpbin.org/delay/2 -> 200 (2.21s)
[✓] https://httpbin.org/delay/2 -> 200 (2.19s)
總耗時: 3.14s   ← 約等於最慢的單個請求，而非加總
```

若使用單執行緒依序下載，總耗時約為 9 秒；使用執行緒池則接近最慢請求（約 3 秒），**加速約 3 倍**。

## 12. 常見陷阱與注意事項

### 12.1 死結 (Deadlock)

兩個執行緒各自持有對方需要的鎖，導致互相等待，永遠無法繼續。

```python
import threading
import time

lock_a = threading.Lock()
lock_b = threading.Lock()

def task_1():
    with lock_a:
        print("[T1] 持有 Lock A")
        time.sleep(0.1)
        with lock_b:   # 此時 lock_b 可能被 T2 持有
            print("[T1] 取得 Lock B")

def task_2():
    with lock_b:
        print("[T2] 持有 Lock B")
        time.sleep(0.1)
        with lock_a:   # 此時 lock_a 可能被 T1 持有
            print("[T2] 取得 Lock A")

t1 = threading.Thread(target=task_1)
t2 = threading.Thread(target=task_2)
t1.start()
t2.start()
t1.join()
t2.join()
# 程式可能永遠不會結束（死結）
```

#### 避免死結的方法

1. **固定鎖定順序** — 所有執行緒以相同順序 acquire
2. **使用 `timeout`** — `lock.acquire(timeout=1)`，逾時則釋放已持有的鎖
3. **使用 `RLock`** — 避免同一執行緒因遞迴而自己鎖死自己

```python
# 解法：全域統一鎖定順序 (先 A 後 B)
def task_1_fixed():
    with lock_a:
        time.sleep(0.1)
        with lock_b:
            print("[T1] 安全取得兩把鎖")

def task_2_fixed():
    with lock_a:   # 必須先 acquire A 再 acquire B
        time.sleep(0.1)
        with lock_b:
            print("[T2] 安全取得兩把鎖")
```

### 12.2 遺漏的例外

執行緒內發生的例外預設**不會自動傳播**到主執行緒。

```python
import threading

def broken():
    raise ValueError("執行緒內部錯誤")

t = threading.Thread(target=broken)
t.start()
t.join()                      # 這裡什麼都不會發生！
print("主程式繼續執行...")     # ← 這行會執行
```

解決方式：自訂 `Thread` 子類別儲存例外，或定期檢查執行緒狀態。

### 12.3 僵持的 join

在 daemon 執行緒上使用 `join()` 可能導致無限等待。

### 12.4 過多的執行緒

每個執行緒約佔 8 MB 記憶體（在大多數平台上），建立數千個執行緒會耗盡資源。應使用 **ThreadPoolExecutor** 限制最大數量。

**題目**：建立一個 daemon 執行緒，每秒列印一次 "心跳檢測中..."，主程式在 3 秒後結束。觀察 daemon 執行緒的行為。

<details>
<summary>解答</summary>

```python
import threading
import time

def heartbeat():
    while True:
        print("心跳檢測中...")
        time.sleep(1)

t = threading.Thread(target=heartbeat, daemon=True)
t.start()
time.sleep(3)
print("主程式結束，daemon 執行緒被強制終止")
```

### 🟡 中等 — 安全的計數器

**題目**：10 個執行緒同時對共用變數 `counter` 累加 50,000 次。使用 `Lock` 確保最終結果等於 `10 × 50,000 = 500,000`。

<details>
<summary>解答</summary>

```python
import threading

counter = 0
lock = threading.Lock()
ITER = 50_000

def add():
    global counter
    for _ in range(ITER):
        with lock:
            counter += 1

threads = [threading.Thread(target=add) for _ in range(10)]
for t in threads: t.start()
for t in threads: t.join()

print(f"結果: {counter} (期望: {ITER * 10})")
```
</details>

### 🟡 中等 — 生產者-消費者

**題目**：一個生產者產生 10 個隨機數字放入 `queue.Queue`，兩個消費者並行取出並計算平方，最後主程式印出所有結果。

<details>
<summary>解答</summary>

```python
import threading
import queue
import random
import time

q = queue.Queue()
results = []
results_lock = threading.Lock()

def producer(count: int):
    for _ in range(count):
        num = random.randint(1, 100)
        q.put(num)
        print(f"[生產者] 放入: {num}")
        time.sleep(random.uniform(0.05, 0.15))
    q.put(None)       # 給消費者 A 的結束信號
    q.put(None)       # 給消費者 B 的結束信號

def consumer(name: str):
    while True:
        item = q.get()
        if item is None:
            q.task_done()
            break
        square = item ** 2
        with results_lock:
            results.append((item, square))
        print(f"[消費者 {name}] {item}² = {square}")
        q.task_done()

prod = threading.Thread(target=producer, args=(10,))
cons_a = threading.Thread(target=consumer, args=("A",))
cons_b = threading.Thread(target=consumer, args=("B",))

prod.start()
cons_a.start()
cons_b.start()
prod.join()
q.join()
cons_a.join()
cons_b.join()

print(f"\n所有結果: {sorted(results)}")
```
</details>

### 🔴 進階 — 避免死結

**題目**：三個執行緒各持有兩把鎖中的一把，造成死結。請修正程式碼，確保所有執行緒都能安全完成。

<details>
<summary>解答</summary>

```python
import threading
import time

lock_a = threading.Lock()
lock_b = threading.Lock()
lock_c = threading.Lock()

# 統一鎖定順序：按照 a → b → c 取得鎖
def worker(name: str, first: threading.Lock, second: threading.Lock):
    print(f"[{name}] 嘗試取得第一把鎖...")
    with first:
        time.sleep(0.05)
        print(f"[{name}] 嘗試取得第二把鎖...")
        with second:
            print(f"[{name}] 完成工作")

# 錯誤版本（可能死結）：
# t1 = threading.Thread(target=worker, args=("T1", lock_a, lock_b))
# t2 = threading.Thread(target=worker, args=("T2", lock_b, lock_a))

# 修正版本：全部先 lock_a 再 lock_b
t1 = threading.Thread(target=worker, args=("T1", lock_a, lock_b))
t2 = threading.Thread(target=worker, args=("T2", lock_a, lock_b))

t1.start()
t2.start()
t1.join()
t2.join()
print("無死結發生！")
```
</details>

### 🔴 進階 — ThreadPoolExecutor 下載器

**題目**：使用 `ThreadPoolExecutor` 下載以下 URL，並使用 `map()` 收集結果。當某個 URL 下載失敗時不能影響其他 URL。

```python
urls = [
    "https://httpbin.org/status/200",
    "https://httpbin.org/status/404",
    "https://httpbin.org/delay/1",
    "https://httpbin.org/status/500",
    "https://httpbin.org/delay/2",
]
```

<details>
<summary>解答</summary>

```python
from concurrent.futures import ThreadPoolExecutor
import urllib.request
import time

urls = [
    "https://httpbin.org/status/200",
    "https://httpbin.org/status/404",
    "https://httpbin.org/delay/1",
    "https://httpbin.org/status/500",
    "https://httpbin.org/delay/2",
]

def safe_fetch(url: str) -> dict:
    try:
        start = time.time()
        with urllib.request.urlopen(url, timeout=5) as resp:
            elapsed = time.time() - start
            return {"url": url, "status": resp.status, "elapsed": elapsed}
    except Exception as e:
        return {"url": url, "error": str(e)}

with ThreadPoolExecutor(max_workers=5) as executor:
    results = executor.map(safe_fetch, urls)

for r in results:
    if "error" in r:
        print(f"[✗] {r['url']} -> 錯誤: {r['error']}")
    else:
        print(f"[✓] {r['url']} -> {r['status']} ({r['elapsed']:.2f}s)")
```
</details>

## 📌 重點整理

| 概念 | 關鍵要點 |
|------|----------|
| **執行緒 vs 行程** | 執行緒共用記憶體、輕量、適合 I/O；行程隔離、重量、適合 CPU |
| **Thread(target=, args=)** | 使用目標函數建立執行緒，`start()` 啟動，`join()` 等待 |
| **Daemon** | `daemon=True` 的主程式結束即終止，不適合需要清理的任務 |
| **競爭危害** | 多執行緒寫入共用變數導致結果不可預測，需 Lock 保護 |
| **Lock / RLock / Semaphore** | Lock 基本互斥；RLock 可重入；Semaphore 控制並發數量 |
| **Event** | `set()` / `wait()` 實現執行緒間單次信號通知 |
| **Queue** | 執行緒安全的 FIFO，內建鎖定，適合生產者-消費者 |
| **GIL** | CPython 限制同一時間只有一個執行緒執行位元組碼 |
| **ThreadPoolExecutor** | 高層次 API，`submit()` + `as_completed()` 或 `map()` |
| **死結** | 固定鎖定順序、使用 timeout、使用 RLock 來避免 |
| **I/O vs CPU** | I/O 密集型用 `threading` 或 `asyncio`；CPU 密集型用 `multiprocessing` |
