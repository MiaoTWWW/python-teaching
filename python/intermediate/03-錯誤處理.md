# 03 - 錯誤與例外處理

> 🎯 **學習目標**：學會處理程式中的錯誤，撰寫更穩健的程式碼。

## 錯誤的類型

### 語法錯誤（SyntaxError）

```python
# ❌ 語法錯誤
# print("Hello"  # 少了右括號
# if True       # 少了冒號
#     print("hi")
```

語法錯誤是**最容易被發現和修正**的錯誤，IDE 和直譯器會直接告訴你。

### 例外（Exception）

例外是在**執行時期**發生的錯誤：

```python
# NameError: 變數未定義
# print(undefined_variable)

# TypeError: 型別不符
# print("數字是: " + 42)

# ValueError: 值不正確
# int("abc")

# IndexError: 索引超出範圍
# numbers = [1, 2, 3]
# print(numbers[10])

# KeyError: 鍵不存在
# student = {"name": "Alice"}
# print(student["age"])

# ZeroDivisionError: 除以零
# result = 10 / 0

# FileNotFoundError: 檔案不存在
# with open("not_exist.txt") as f:
#     content = f.read()
```

## try / except 語法

### 基本用法

```python
try:
    number = int(input("請輸入一個數字: "))
    result = 100 / number
    print(f"結果: {result}")
except ValueError:
    print("錯誤：請輸入有效的數字！")
except ZeroDivisionError:
    print("錯誤：不能除以零！")
```

### 捕捉多種例外

```python
try:
    numbers = [1, 2, 3]
    index = int(input("請輸入索引: "))
    print(numbers[index])
except (ValueError, IndexError) as e:
    print(f"發生錯誤: {e}")
```

### 捕捉所有例外（不建議）

```python
try:
    # 一些可能出錯的程式碼
    result = risky_operation()
except Exception as e:
    print(f"發生未預期的錯誤: {e}")
    # 記錄錯誤日誌
    # 進行復原操作
```

> ⚠️ 捕捉所有例外應該**謹慎使用**，最好還是針對特定例外處理。

## else 與 finally

```python
try:
    file = open("data.txt", "r", encoding="utf-8")
    content = file.read()
except FileNotFoundError:
    print("檔案不存在")
else:
    # 沒有例外發生時執行
    print(f"成功讀取檔案，共 {len(content)} 字元")
finally:
    # 無論如何都會執行（適合清理資源）
    if 'file' in locals():
        file.close()
        print("檔案已關閉")
```

### 實際應用：使用 with 語句

```python
# 使用 with 自動管理資源（不用手動 close）
try:
    with open("data.txt", "r", encoding="utf-8") as file:
        content = file.read()
        print(content)
except FileNotFoundError:
    print("找不到檔案")
except PermissionError:
    print("沒有讀取權限")
```

## 引發例外

### raise 語句

```python
def withdraw(balance, amount):
    if amount <= 0:
        raise ValueError("提領金額必須大於 0")
    if amount > balance:
        raise ValueError("餘額不足")
    return balance - amount

# 使用
try:
    new_balance = withdraw(1000, 2000)
except ValueError as e:
    print(f"交易失敗: {e}")
```

### 自訂例外類別

```python
class BankError(Exception):
    """銀行系統的基礎例外"""
    pass

class InsufficientBalanceError(BankError):
    """餘額不足例外"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"餘額不足：目前 ${balance}，需要 ${amount}")

class InvalidAmountError(BankError):
    """無效金額例外"""
    def __init__(self, amount):
        self.amount = amount
        super().__init__(f"無效的金額：${amount}")

class Account:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def withdraw(self, amount):
        if amount <= 0:
            raise InvalidAmountError(amount)
        if amount > self.balance:
            raise InsufficientBalanceError(self.balance, amount)
        self.balance -= amount
        return self.balance

# 使用自訂例外
account = Account("Alice", 1000)

try:
    account.withdraw(2000)
except InsufficientBalanceError as e:
    print(f"❌ {e}")
except InvalidAmountError as e:
    print(f"❌ {e}")
except BankError as e:
    print(f"❌ 銀行系統錯誤: {e}")
else:
    print(f"✅ 交易成功，餘額: {account.balance}")
```

## 例外處理最佳實踐

### 1. 不要忽略例外

```python
# ❌ 錯誤：空的 except
try:
    risky_operation()
except Exception:
    pass  # 忽略所有錯誤！

# ✅ 正確：至少要記錄
try:
    risky_operation()
except Exception as e:
    print(f"發生錯誤: {e}")
    # 或 logging.exception("發生錯誤")
```

### 2. 盡量具體

```python
# ✅ 好：明確捕捉特定例外
try:
    result = int(input("輸入數字: "))
except ValueError:
    print("請輸入有效的數字")
```

### 3. 不要用例外來控制流程

```python
# ❌ 錯誤使用例外
numbers = [1, 2, 3]
try:
    i = 0
    while True:
        print(numbers[i])
        i += 1
except IndexError:
    pass

# ✅ 使用正常流程控制
for num in numbers:
    print(num)
```

### 4. 使用 EAFP 風格

Python 鼓勵 **EAFP**（Easier to Ask for Forgiveness than Permission）：

```python
# LBYL（Look Before You Leap）- 不建議
if os.path.exists("data.txt"):
    with open("data.txt") as f:
        content = f.read()

# EAFP（請求原諒比允許容易）- 建議
try:
    with open("data.txt") as f:
        content = f.read()
except FileNotFoundError:
    print("檔案不存在")
```

## 實用範例

### 安全的數值輸入

```python
def get_number(prompt):
    """安全地取得使用者輸入的數字"""
    while True:
        try:
            return float(input(prompt))
        except ValueError:
            print("請輸入有效的數字！")

# 使用
age = get_number("請輸入年齡: ")
print(f"你輸入的年齡是: {age}")
```

### 斷言（assert）

```python
def divide(a, b):
    """安全的除法"""
    assert b != 0, "除數不能為 0"
    return a / b

# 使用
try:
    result = divide(10, 0)
except AssertionError as e:
    print(f"斷言錯誤: {e}")
```

## 重點整理

| 概念 | 關鍵字/用法 |
|------|------------|
| 例外處理 | `try` / `except` / `else` / `finally` |
| 引發例外 | `raise` |
| 自訂例外 | 繼承 `Exception` |
| 資源管理 | `with` 語句 |
| 快速除錯 | `assert` |

## 🏆 經典練習題

### try/except 結構 🟢 簡易 | 🟡 中等

1. **安全除法**：寫一個函式 `safe_divide(a, b)`，當 b 為 0 時捕捉 `ZeroDivisionError` 並回傳 `None`
   ```python
   print(safe_divide(10, 2))  # 5.0
   print(safe_divide(10, 0))  # None
   ```
   💡 提示：`try: return a / b; except ZeroDivisionError: return None`

2. **數字輸入**：用 `try/except` 確保使用者輸入的是有效整數，否則一直要求重新輸入
   💡 提示：`while True` + `try: int(input())` + `except ValueError`

3. **else 子句**：寫一個範例展示 `try/except/else` 的執行流程 — 只有在 except 沒被觸發時 else 才會執行
   💡 提示：else 在 try 成功後執行

### 多種例外處理 🟡 中等

4. **多層捕捉**：一段程式可能拋出 `ValueError`、`FileNotFoundError`、`PermissionError`，用多個 `except` 分別處理
   💡 提示：依特定 → 一般順序排列 except

5. **例外配對**：建立一個字典來對應不同的例外型別與處理訊息
   ```python
   error_handlers = {
       ValueError: "輸入格式錯誤",
       ZeroDivisionError: "除以零",
   }
   ```
   💡 提示：`type(e).__name__` 可以取得例外名稱

### 自訂例外類別 🟡 中等 | 🔴 困難

6. **密碼驗證**：建立 `PasswordError` 自訂例外，以及 `validate_password(pwd)` 函式（至少 8 碼、含數字和字母）
   ```python
   class PasswordError(Exception):
       pass
   ```
   💡 提示：不滿足條件時 `raise PasswordError("原因")`

7. **餘額不足**：建立 `InsufficientFundsError`，在 `BankAccount.withdraw()` 中餘額不足時拋出
   💡 提示：自訂例外可攜帶額外資訊，如 `self.balance`

8. **多級例外**：建立一個 `ApplicationError` → `DatabaseError` → `ConnectionError` 的例外繼承體系
   💡 提示：用 `except ApplicationError` 可一次捕捉所有子類別例外

### finally 清理 🟡 中等

9. **資源清理**：用 `try/finally` 確保檔案在讀取後一定會被關閉（不使用 `with` 陳述式）
   💡 提示：`finally` 在 `return` 之後仍會執行

10. **函式返回順序**：寫一個函式，在 try 中 return，觀察 finally 是否仍在 return 之前執行？
    💡 提示：finally 一定會執行，甚至在 return 之前

### assert 除錯 🟢 簡易 | 🟡 中等

11. **輸入驗證**：用 `assert` 在函式開頭檢查參數是否合法
    ```python
    def set_age(age):
        assert 0 <= age <= 150, f"年齡 {age} 超出合理範圍"
    ```
    💡 提示：assert 失敗拋出 `AssertionError`

12. **不變性檢查**：在函式執行的關鍵點用 `assert` 驗證程式的假設是否成立
    💡 提示：assert 在 `python -O` 模式下會被忽略

### with 與 context manager 🟡 中等 | 🔴 困難

13. **自訂 context manager**：建立 `Timer` context manager，用 `with Timer():` 計算區塊內程式執行時間
    ```python
    with Timer("資料處理"):
        # 執行一些操作
        time.sleep(0.5)
    # 輸出：資料處理 耗時 0.50 秒
    ```
    💡 提示：實作 `__enter__` 和 `__exit__`

14. **抑制例外**：在 `__exit__` 中回傳 `True`，讓 context manager 吞掉例外
    💡 提示：回傳 `True` 表示例外已被處理

15. **contextlib 簡化**：使用 `@contextmanager` 裝飾器改寫自訂 context manager
    ```python
    from contextlib import contextmanager
    @contextmanager
    def my_context():
        # 設定
        yield
        # 清理
    ```
    💡 提示：`yield` 前後的程式碼分別對應 `__enter__` 和 `__exit__`
