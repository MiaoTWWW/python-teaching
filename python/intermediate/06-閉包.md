# 06 - 閉包（Closure）

> 🎯 **學習目標**：理解閉包的運作機制，掌握內部函式捕獲外部變數的技術以及 `nonlocal` 關鍵字的使用。

## 什麼是閉包？

**閉包（Closure）** 是一個函式「記住」了它被定義時所在作用域的變數，即使那個作用域已經執行完畢。

簡單說：**閉包 = 函式 + 捕獲的自由變數**

```python
def outer(msg):
    def inner():
        print(msg)  # inner 記住了 msg
    return inner

closure_func = outer("Hello!")
closure_func()  # Hello!
```

當 `outer("Hello!")` 執行完畢後，區域變數 `msg` 理論上應該消失，但 `inner` 仍然可以存取它——這就是閉包的力量。

## 閉包的三個必要條件

一個函式要形成閉包，必須**同時滿足**三個條件：

```python
def outer(x):           # ① 有外部函式
    def inner(y):       # ② 內部函式（巢狀定義）
        return x + y    # ③ 內部函式參考了外部函式的變數
    return inner        # ④ 外部函式回傳內部函式

add5 = outer(5)
print(add5(3))  # 8
print(add5(10)) # 15
```

| 條件 | 說明 |
|------|------|
| ① 函式巢狀 | 一個函式定義在另一個函式內部 |
| ② 參考外部變數 | 內部函式使用了外部函式的變數 |
| ③ 外部回傳內部 | 外部函式將內部函式作為回傳值 |

這三點缺一不可，否則就不算是閉包。

## 檢查閉包：`__closure__`

Python 提供 `__closure__` 屬性來檢查一個函式是否為閉包：

```python
def outer(x):
    def inner(y):
        return x + y
    return inner

add5 = outer(5)
print(add5.__closure__)   # (<cell at 0x...: int object at 0x...>,)
print(add5.__closure__[0].cell_contents)  # 5

# 非閉包的情況
def normal_func(x):
    return x + 1

print(normal_func.__closure__)  # None
```

`__closure__` 是一個由 **cell 物件**組成的元組，每個 cell 物件對應一個被捕獲的自由變數：

```python
def capture_multiple(a, b):
    def inner():
        return a + b
    return inner

fn = capture_multiple(10, 20)
print(len(fn.__closure__))           # 2（兩個變數）
print(fn.__closure__[0].cell_contents)  # 10
print(fn.__closure__[1].cell_contents)  # 20
```

## 閉包的運作機制

當 Python 看到一個巢狀函式參考了外部變數時，它會：

1. 在編譯時期發現 `inner` 使用了 `x`
2. 將 `x` 標記為「自由變數」（free variable）
3. 建立 cell 物件來存放 `x`
4. 即使 `outer` 結束，cell 物件仍然存活，讓 `inner` 可以繼續存取

```python
def outer():
    count = 0
    def inner():
        nonlocal count
        count += 1
        return count
    return inner

counter = outer()
print(counter())  # 1
print(counter())  # 2
print(counter())  # 3

# 查看 cell 物件的變化
print(counter.__closure__[0].cell_contents)  # 3
```

## `nonlocal` 關鍵字

內部函式預設只能**讀取**外部變數，不能**重新賦值**。要修改外部變數，需要使用 `nonlocal` 宣告：

```python
def outer():
    count = 0
    def inner():
        # 沒有 nonlocal 會怎樣？
        # count += 1  # UnboundLocalError！
        nonlocal count  # ✅ 告訴 Python count 來自外層作用域
        count += 1
        return count
    return inner
```

`nonlocal` 與 `global` 的區別：

```python
x = "全域"

def outer():
    x = "外層"

    def inner():
        global x    # 指向全域的 x
        x = "改了全域"
        print(x)

    inner()
    print(x)  # "外層"（outer 的 x 沒變）

outer()
print(x)  # "改了全域"
```

| 關鍵字 | 作用範圍 |
|--------|----------|
| `global` | 指向模組層級的全域變數 |
| `nonlocal` | 指向最近的外層函式區域變數（不含全域） |

## 閉包的實戰應用

### 1. 工廠函式（Factory Function）

根據不同參數產生不同行為的函式：

```python
def make_multiplier(factor):
    """建立倍數乘法器"""
    def multiply(number):
        return number * factor
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(10))   # 20
print(triple(10))   # 30
print(double(7))    # 14

# 實用範例：單位轉換
def make_converter(ratio, offset=0):
    def convert(value):
        return value * ratio + offset
    return convert

c_to_f = make_converter(9/5, 32)    # 攝氏轉華氏
m_to_km = make_converter(1/1000)     # 公尺轉公里

print(c_to_f(100))   # 212.0
print(m_to_km(1500)) # 1.5
```

### 2. 計數器工廠

```python
def make_counter(start=0):
    """建立一個獨立的計數器"""
    count = start
    def counter():
        nonlocal count
        current = count
        count += 1
        return current
    return counter

# 各自獨立，互不干擾
counter_a = make_counter(1)
counter_b = make_counter(100)

print(counter_a())  # 1
print(counter_a())  # 2
print(counter_b())  # 100
print(counter_b())  # 101
print(counter_a())  # 3（不影響 b）
```

### 3. 資料隱藏（封裝）

閉包可以模擬私有變數的行為：

```python
def create_account(initial_balance=0):
    """建立銀行帳戶（使用閉包隱藏餘額）"""
    balance = initial_balance

    def deposit(amount):
        nonlocal balance
        if amount <= 0:
            raise ValueError("存款金額必須為正數")
        balance += amount
        return balance

    def withdraw(amount):
        nonlocal balance
        if amount <= 0:
            raise ValueError("提款金額必須為正數")
        if amount > balance:
            raise ValueError("餘額不足")
        balance -= amount
        return balance

    def get_balance():
        return balance

    # 只回傳公開的操作介面
    return {"deposit": deposit, "withdraw": withdraw, "balance": get_balance}

# 使用
account = create_account(1000)
account["deposit"](500)
account["withdraw"](200)
print(account["balance"]())  # 1300

# balance 無法從外部直接存取！
# print(account["balance"])  # 這是函式，不是數值
# account.balance  # ❌ 根本不存在這個屬性
```

### 4. 回呼函式（Callback）生成器

```python
# 建立按鈕點擊處理器（模擬 GUI 事件）
def make_button_handler(button_name):
    def handler(event):
        print(f"[{button_name}] 被點擊了！事件：{event}")

btn_save = make_button_handler("儲存")

btn_cancel = make_button_handler("取消")

```

### 5. 延遲計算與快取

```python
def make_lazy_adder(x):
    """延遲加法：先記住 x，等真正需要時再算"""
    def adder(y):
        return x + y
    return adder

add10 = make_lazy_adder(10)
# 中間可以做其他事情...
result = add10(25)
print(result)  # 35

# 簡單的快取
def make_cache(func):
    cache = {}
    def wrapper(key):
        nonlocal cache
        if key not in cache:
            cache[key] = func(key)
            print(f"  計算: {key}")
        else:
            print(f"  快取: {key}")
        return cache[key]
    return wrapper

@make_cache  # 第 7 章會詳細說明裝飾器語法
def expensive_func(n):
    return n * n

# 暫時手動包裝
cached_square = make_cache(lambda n: n * n)
print(cached_square(5))  # 計算: 5 → 25
print(cached_square(5))  # 快取: 5 → 25
print(cached_square(10)) # 計算: 10 → 100
```

## 閉包 vs 全域變數 vs 類別

同一功能用三種方式實作比較：

```python
# ─── 方式一：全域變數 ───
count_global = 0
def increment_global():
    global count_global
    count_global += 1
    return count_global

print(increment_global())  # 1
print(increment_global())  # 2
# 缺點：任何地方都能修改 count_global，不安全

# ─── 方式二：類別 ───
class Counter:
    def __init__(self):
        self.count = 0
    def increment(self):
        self.count += 1
        return self.count

c = Counter()
print(c.increment())  # 1
print(c.increment())  # 2
# 優點：完整 OOP，可繼承
# 缺點：語法較重，用於簡單計數略嫌繁瑣

# ─── 方式三：閉包 ───
def make_counter_simple():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

inc = make_counter_simple()
print(inc())  # 1
print(inc())  # 2
# 優點：輕量、專注、變數完全隱藏
# 缺點：功能有限，無法像類別那樣擴展
```

| 特性 | 全域變數 | 類別 | 閉包 |
|------|----------|------|------|
| 語法輕量 | ✅ | ❌ | ✅ |
| 資料隱藏 | ❌ | ✅（私有屬性） | ✅ |
| OOP 擴展 | ❌ | ✅（繼承） | ❌ |
| 記憶體 | ✅ | ✅ | ⚠️ 輕量 |
| 適合場景 | 簡單全域狀態 | 複雜狀態 + 行為 | 單一函式、工廠模式 |

## 閉包的常見陷阱

### 陷阱一：延遲綁定（Late Binding）

最經典的閉包陷阱——在迴圈中建立閉包：

```python
# ❌ 問題：所有函式都輸出相同的值
funcs = []
for i in range(5):
    funcs.append(lambda: i)

for f in funcs:
    print(f(), end=" ")  # 4 4 4 4 4（不是 0 1 2 3 4！）
```

為什麼？因為 `lambda: i` 中的 `i` 是個自由變數，它在**被呼叫時**才去查找 `i` 的值。當迴圈結束時，`i` 已經變成 4 了，所以所有函式都輸出 4。

```python
# 用 __closure__ 驗證
funcs = []
for i in range(5):
    funcs.append(lambda: i)

for f in funcs:
    print(f.__closure__[0].cell_contents, end=" ")
# 4 4 4 4 4（所有 cell 都指向同一個 i！）
```

### 解法一：立即綁定（預設引數）

利用 Python 的預設引數在**定義時期**就會被求值的特性：

```python
# ✅ 解法：使用預設引數立即綁定目前的值
funcs = []
for i in range(5):
    funcs.append(lambda x=i: x)  # 傳入預設值

for f in funcs:
    print(f(), end=" ")  # 0 1 2 3 4 ✅
```

為什麼有效？`x=i` 在 `lambda` 被建立時就求值了，`i` 的當下值被固定在 `x` 中。此時 `x` 是區域變數（參數），不是自由變數，所以不會有延遲綁定的問題。

```python
# 驗證：現在每個 cell 持有不同的值
funcs = []
for i in range(5):
    funcs.append(lambda x=i: x)

for f in funcs:
    print(f.__closure__, end=" ")
# None None None None None（沒有閉包！因為 x 是參數）
```

### 解法二：中層函式（工廠函式）

```python
# ✅ 解法：用一個中層函式建立新的作用域
def make_lambda(x):
    return lambda: x

funcs = []
for i in range(5):
    funcs.append(make_lambda(i))

for f in funcs:
    print(f(), end=" ")  # 0 1 2 3 4 ✅
```

每次呼叫 `make_lambda(i)` 都會建立新的作用域，`x` 的值被固定在該作用域中。

### 陷阱二：修改可變物件

```python
# ⚠️ 這是合理的行為，不是 bug
adders = []
for i in range(5):
    adders.append(lambda x, y=i: x + y)

print(adders[0](10))  # 10
print(adders[3](10))  # 13
```

```python
# ❌ 錯誤：所有按鈕都列印最後一個名稱
```

## 綜合實例

### 實例一：可自訂步進的計數器

```python
def make_stepped_counter(start=0, step=1):
    """建立可自訂起始值與步進值的計數器"""
    count = start
    def increment():
        nonlocal count
        current = count
        count += step
        return current
    def reset():
        nonlocal count
        count = start
    def current():
        return count
    return {"next": increment, "reset": reset, "current": current}

counter = make_stepped_counter(10, 5)
print(counter["next"]())    # 10
print(counter["next"]())    # 15
print(counter["next"]())    # 20
counter["reset"]()
print(counter["next"]())    # 10
```

### 實例二：函式呼叫次數統計

```python
def counted(func):
    """裝飾器風格的呼叫次數統計（閉包實作）"""
    count = 0
    def wrapper(*args, **kwargs):
        nonlocal count
        count += 1
        print(f"呼叫第 {count} 次")
        return func(*args, **kwargs)
    wrapper.get_count = lambda: count  # 附加查詢介面
    return wrapper

def say_hello(name):
    return f"哈囉，{name}"

say_hello = counted(say_hello)  # 手動套用

print(say_hello("Alice"))
print(say_hello("Bob"))
print(say_hello("Charlie"))
print(f"總共呼叫了 {say_hello.get_count()} 次")
# 呼叫第 1 次
# 哈囉，Alice
# 呼叫第 2 次
# 哈囉，Bob
# 呼叫第 3 次
# 哈囉，Charlie
# 總共呼叫了 3 次
```

## 重點整理

| 概念 | 說明 |
|------|------|
| 閉包（Closure） | 函式 + 記住的外部變數 |
| 三個必要條件 | 巢狀函式 + 參考外部變數 + 外部回傳內部 |
| `__closure__` | 儲存被捕捉的自由變數的 cell 物件元組 |
| `nonlocal` | 修改外層函式的區域變數 |
| 延遲綁定陷阱 | 迴圈中建立的閉包都共享同一個變數 |
| 預設引數解法 | `lambda x=i: x` 立即綁定目前值 |
| 工廠函式解法 | 用中層函式建立獨立作用域 |

1. **簡單問候**：寫一個 `greeter` 函式，接受一個問候語（如 "你好"），回傳一個函式接受名字並印出完整問候
   ```python
   hello = greeter("你好")

   hi = greeter("嗨")
   hi("小華")    # 嗨，小華！
   ```
   💡 提示：內部函式 `def inner(name): print(f"{greeting}，{name}!")`

2. **倍數檢查器**：寫一個 `make_divisible_checker(n)`，回傳一個函式檢查數字是否為 n 的倍數

   ```python
   is_even = make_divisible_checker(2)
   print(is_even(10))  # True
   print(is_even(7))   # False
   is_multiple_of_3 = make_divisible_checker(3)
   print(is_multiple_of_3(9))  # True
   ```
   💡 提示：回傳 `lambda x: x % n == 0`

3. **字首過濾器**：寫一個 `make_prefix_filter(prefix)`，回傳一個函式判斷字串是否以指定字首開頭

   ```python
   print(starts_with_http("ftp://files.com"))      # False
   ```
   💡 提示：內部使用 `s.startswith(prefix)`
