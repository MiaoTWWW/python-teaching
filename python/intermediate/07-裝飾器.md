## 1. 什麼是裝飾器？

**裝飾器（Decorator）** 是一種用來「修改或增強函式行為」而不改變函式原始程式碼的手法。

本質上，裝飾器就是一個**高階函式（higher-order function）**：

1. 接收一個函式作為參數
2. 在內部定義一個新的包裝函式
3. 回傳這個包裝函式

```python
def decorator(func):
    def wrapper():
        print("—— 呼叫前 ——")
        func()
        print("—— 呼叫後 ——")
    return wrapper
```

## 2. `@` 語法 vs 手動裝飾

### 手動裝飾

```python
def say_hello():
    print("你好！")

say_hello = decorator(say_hello)
say_hello()
```

### `@` 語法糖

```python
@decorator
def say_hello():
    print("你好！")

say_hello()
```

兩者完全等價。`@decorator` 只是 Python 提供的語法糖，編譯器會自動展開成 `say_hello = decorator(say_hello)`。

```python
def decorator(func):
    def wrapper():
        print("—— 呼叫前 ——")
        func()
        print("—— 呼叫後 ——")
    return wrapper

@decorator
def say_hello():
    print("你好！")

say_hello()
# 輸出:
# —— 呼叫前 ——
# 你好！
# —— 呼叫後 ——
```

## 3. 第一個實戰：Timer 計時器

```python
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} 耗時 {elapsed:.4f} 秒")
        return result
    return wrapper

@timer
def slow_add(a, b):
    time.sleep(0.5)
    return a + b

print(slow_add(3, 5))
# 輸出:
# slow_add 耗時 0.5003 秒
# 8
```

注意到 `wrapper(*args, **kwargs)` — 這是讓裝飾器可以適配任意簽名（signature）的關鍵寫法。

## 4. `@functools.wraps` — 保護元資料

上面的裝飾器有個問題：被裝飾的函式會「失去身份」。

```python
print(slow_add.__name__)  # wrapper  ← 不是 slow_add！
print(slow_add.__doc__)   # None    ← 文件字串也不見了
```

解決方式：`@functools.wraps`

```python
import functools
import time

def timer(func):
    @functools.wraps(func)          # ← 把 func 的元資料複製到 wrapper
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} 耗時 {elapsed:.4f} 秒")
        return result
    return wrapper

@timer
def slow_add(a, b):
    """回傳 a 與 b 的和，刻意延遲 0.5 秒"""
    time.sleep(0.5)
    return a + b

print(slow_add.__name__)  # slow_add  ← 正確！
print(slow_add.__doc__)   # 回傳 a 與 b 的和... ← 正確！
```

**規則**：**每個裝飾器內部都應該使用 `@functools.wraps`**，除非你有非常明確的理由不這麼做。

## 5. 裝飾器帶參數（三層巢狀）

如果希望裝飾器本身接受參數（例如指定 prefix），需要**多一層外殼**：

```python
import functools

def prefix_log(prefix):                  # 第一層：接收裝飾器參數
    def decorator(func):                 # 第二層：接收函式（這是真正的裝飾器）
        @functools.wraps(func)
        def wrapper(*args, **kwargs):    # 第三層：接收原函式參數
            print(f"[{prefix}] 呼叫 {func.__name__}")
            return func(*args, **kwargs)
        return wrapper
    return decorator

@prefix_log("API")
def fetch_data(url):
    return f"資料來自 {url}"

print(fetch_data("https://example.com"))
# 輸出:
# [API] 呼叫 fetch_data
# 資料來自 https://example.com
```

等價於：

```python
fetch_data = prefix_log("API")(fetch_data)
```

## 6. 多個裝飾器堆疊順序

**裝飾器由下往上疊加，執行時由上往下**。

```python
def bold(func):
    @functools.wraps(func)
    def wrapper():
        return f"<b>{func()}</b>"
    return wrapper

def italic(func):
    @functools.wraps(func)
    def wrapper():
        return f"<i>{func()}</i>"
    return wrapper

@bold          # 第二步：bold 包裝 italic 的結果
@italic        # 第一步：italic 先包裝 hello
def hello():
    return "哈囉"

print(hello())  # <b><i>哈囉</i></b>
```

展開等同於：

```python
hello = bold(italic(hello))
```

**重點**：離函式最近的 `@` 最先被套用。

## 7. 實用裝飾器範例

### 7.1 `@timer` — 執行時間

```python
import functools
import time

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"⌛ {func.__name__} 耗時 {elapsed:.4f}s")
        return result
    return wrapper
```

### 7.2 `@debug` — 印出呼叫參數與回傳值

```python
def debug(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        args_repr = [repr(a) for a in args]
        kwargs_repr = [f"{k}={v!r}" for k, v in kwargs.items()]
        signature = ", ".join(args_repr + kwargs_repr)
        print(f"🔍 呼叫 {func.__name__}({signature})")
        result = func(*args, **kwargs)
        print(f"🔍 回傳 {func.__name__}({signature}) = {result!r}")
        return result
    return wrapper

@debug
def multiply(x, y):
    return x * y

multiply(3, 4)
# 輸出:
# 🔍 呼叫 multiply(3, 4)
# 🔍 回傳 multiply(3, 4) = 12
```

### 7.3 `@retry` — 失敗重試

```python
import functools
import time
import random

def retry(max_attempts=3, delay=1):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"⚠️ 第 {attempt} 次失敗：{e}")
                    if attempt == max_attempts:
                        raise
                    time.sleep(delay)
            return None
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.5)
def unstable_api():
    if random.random() < 0.7:
        raise ConnectionError("網路不穩")
    return "✅ 成功"

print(unstable_api())
```

### 7.4 `@cache` — 記憶化

```python
import functools

def cache(func):
    store = {}
    @functools.wraps(func)
    def wrapper(*args):
        if args in store:
            print(f"🔄 快取命中：{args}")
            return store[args]
        result = func(*args)
        store[args] = result
        return result
    return wrapper

@cache
def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(10))  # 55
```

> 實務上直接用 `@functools.lru_cache` 更穩健。

### 7.5 `@login_required` — 模擬權限檢查

```python
import functools

def login_required(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        if not getattr(wrapper, "user", None):
            raise PermissionError("請先登入")
        return func(*args, **kwargs)
    return wrapper

@login_required
def view_profile():
    return "個人資料頁面"

# view_profile()            # 💥 PermissionError: 請先登入
login_required.user = "Alice"
print(view_profile())       # 個人資料頁面
```

## 8. 類別型裝飾器（Class-based Decorator）

透過 `__call__` 讓類別實例可以當作裝飾器使用。

```python
import functools

class CountCalls:
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"📞 第 {self.count} 次呼叫 {self.func.__name__}")
        return self.func(*args, **kwargs)

@CountCalls
def greet(name):
    return f"你好，{name}"

print(greet("小明"))  # 📞 第 1 次呼叫 greet / 你好，小明
print(greet("小華"))  # 📞 第 2 次呼叫 greet / 你好，小華
print(f"總共呼叫了 {greet.count} 次")  # 總共呼叫了 2 次
```

> 類別裝飾器適合需要**維護狀態**的場合，並且直覺上比巢狀函式易讀。

也可以寫成**不接收函式、回傳包裝類別**的形式：

```python
class Timer:
    def __init__(self, label="預設"):
        self.label = label

    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            start = time.perf_counter()
            result = func(*args, **kwargs)
            elapsed = time.perf_counter() - start
            print(f"[{self.label}] {func.__name__} 耗時 {elapsed:.4f}s")
            return result
        return wrapper

@Timer(label="資料庫")
def query():
    time.sleep(0.2)
    return "結果"

query()
```

## 9. 常見陷阱

### 陷阱 1：忘記 `@functools.wraps`

```python
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@bad_decorator
def f():
    pass

print(f.__name__)  # wrapper ❌
```

### 陷阱 2：裝飾器只在載入時執行一次

```python
@timer
def heavy():
    pass
# ↑ timer(heavy) 在 import 時就呼叫了，不是 heavy() 被呼叫時
```

### 陷阱 3：裝飾可變預設值

```python
def append_item(item, lst=[]):   # ❌ 可變預設值
    lst.append(item)
    return lst
```

如果裝飾器包裝了這種函式，問題不會消失，只會更難除錯。

### 陷阱 4：類別方法裝飾器忘記 `self`

```python
def method_decorator(func):
    @functools.wraps(func)
    def wrapper(self, *args, **kwargs):   # ← 記得包含 self
        print(f"在類別中呼叫 {func.__name__}")
        return func(self, *args, **kwargs)
    return wrapper
```

### 陷阱 5：兩個裝飾器共用外部變數

```python
def repeat(n):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                func(*args, **kwargs)
        return wrapper
    return decorator

@repeat(3)
def say(word):
    print(word)

say("嗨")  # 嗨 嗨 嗨
```

這段沒問題，但如果你不小心在迴圈中建立多個裝飾器，注意閉包變數的延遲綁定（late binding）。

## 📌 重點整理

| 概念 | 說明 |
|------|------|
| 裝飾器本質 | 接收函式、回傳函式的高階函式 |
| `@` 語法 | `@dec` 等於 `func = dec(func)` |
| `functools.wraps` | 複製 `__name__`、`__doc__` 等元資料，**必加** |
| 帶參數裝飾器 | 三層巢狀：外層收參數 → 中層收函式 → 內層收原參數 |
| 堆疊順序 | `@A` 在上、`@B` 在下 → `A(B(func))` |
| 類別裝飾器 | 實作 `__call__`，適合維護狀態 |
| 實務常見 | `@timer`、`@debug`、`@retry`、`@lru_cache`、`@login_required` |

> 💡 **一句話記住裝飾器**：它是在函式定義完成後，額外包裹一層邏輯又不動到原始碼的工具。
