# 01 - 物件導向程式設計

> 🎯 **學習目標**：理解物件導向程式設計（OOP）的核心概念，並能在 Python 中實踐。

## 什麼是物件導向？

物件導向程式設計是一種**將資料與行為封裝在一起**的程式設計典範。核心概念包括：

- **類別（Class）**：物件的藍圖
- **物件（Object）**：類別的實例
- **封裝（Encapsulation）**：隱藏內部實作細節
- **繼承（Inheritance）**：從現有類別建立新類別
- **多型（Polymorphism）**：相同的介面，不同的實作

## 類別與物件

### 定義類別

```python
class Dog:
    """狗狗類別"""
    # 類別屬性（所有實例共享）
    species = "Canis familiaris"

    # 初始化方法（建構式）
    def __init__(self, name, age):
        self.name = name    # 實例屬性
        self.age = age

    # 實例方法
    def bark(self):
        return f"{self.name} 說: 汪！"

    def __str__(self):
        return f"{self.name} ({self.age} 歲)"
```

### 建立物件

```python
# 建立實例
dog1 = Dog("Buddy", 3)
dog2 = Dog("Lucy", 5)

# 存取屬性
print(dog1.name)          # Buddy
print(dog1.species)       # Canis familiaris
print(Dog.species)        # Canis familiaris（透過類別存取）

# 呼叫方法
print(dog1.bark())        # Buddy 說: 汪！
print(dog2.bark())        # Lucy 說: 汪！

# 使用 __str__
print(dog1)               # Buddy (3 歲)
```

## 封裝（Encapsulation）

Python 使用**命名慣例**來表示存取權限：

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner      # 公開屬性
        self._balance = balance  # 受保護屬性（慣例）
        self.__pin = "1234"     # 私有屬性（名稱修飾）

    # Getter
    @property
    def balance(self):
        return self._balance

    # Setter
    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("餘額不能為負數")
        self._balance = value

    def deposit(self, amount):
        if amount > 0:
            self._balance += amount
            return f"存入 {amount}，餘額: {self._balance}"
        return "金額必須大於 0"

    def withdraw(self, amount):
        if 0 < amount <= self._balance:
            self._balance -= amount
            return f"提領 {amount}，餘額: {self._balance}"
        return "餘額不足或金額無效"

# 使用
account = BankAccount("Alice", 1000)
print(account.balance)           # 1000（使用 getter）
account.balance = 2000           # 使用 setter
print(account.deposit(500))      # 存入 500，餘額: 2500

# 注意：Python 沒有真正的私有
print(account._balance)          # 可以存取（但不建議）
```

## 繼承（Inheritance）

```python
# 父類別
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("子類別必須實作此方法")

    def __str__(self):
        return self.name

# 子類別
class Dog(Animal):
    def speak(self):
        return f"{self.name} 說: 汪！"

class Cat(Animal):
    def speak(self):
        return f"{self.name} 說: 喵～"

class Duck(Animal):
    def speak(self):
        return f"{self.name} 說: 呱！"

# 多型
animals = [Dog("Buddy"), Cat("Kitty"), Duck("Donald")]

for animal in animals:
    print(animal.speak())
# Buddy 說: 汪！
# Kitty 說: 喵～
# Donald 說: 呱！
```

### super() - 呼叫父類別方法

```python
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

    def work(self):
        return f"{self.name} 正在工作"

class Manager(Employee):
    def __init__(self, name, salary, department):
        # 呼叫父類別的 __init__
        super().__init__(name, salary)
        self.department = department

    def work(self):
        # 擴充父類別的方法
        base_work = super().work()
        return f"{base_work}，管理 {self.department} 部門"

    def manage(self):
        return f"{self.name} 正在管理團隊"

# 使用
manager = Manager("Alice", 80000, "工程")
print(manager.work())     # Alice 正在工作，管理 工程 部門
print(manager.manage())   # Alice 正在管理團隊
```

## 魔術方法（Magic Methods）

特殊方法讓你的類別能夠與 Python 內建操作整合：

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"Vector({self.x}, {self.y})"

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)

    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    def __abs__(self):
        return (self.x ** 2 + self.y ** 2) ** 0.5

# 使用
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1 + v2)       # Vector(4, 6)
print(v1 - v2)       # Vector(2, 2)
print(v1 * 3)        # Vector(9, 12)
print(v1 == Vector(3, 4))  # True
print(abs(v1))       # 5.0
```

### 常用魔術方法

| 方法 | 用途 | 觸發時機 |
|------|------|---------|
| `__init__` | 初始化 | `obj = Class()` |
| `__str__` | 字串表示 | `print(obj)`、`str(obj)` |
| `__repr__` | 開發者表示 | `repr(obj)`、除錯 |
| `__len__` | 長度 | `len(obj)` |
| `__getitem__` | 索引存取 | `obj[key]` |
| `__setitem__` | 索引賦值 | `obj[key] = value` |
| `__call__` | 呼叫物件 | `obj()` |
| `__enter__`/`__exit__` | 內容管理器 | `with obj:` |

## 類別方法與靜態方法

```python
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    # 實例方法
    def display(self):
        return f"{self.year}/{self.month:02d}/{self.day:02d}"

    # 類別方法（接收 cls 作為第一個參數）
    @classmethod
    def from_string(cls, date_str):
        """從字串 'YYYY-MM-DD' 建立 Date 物件"""
        year, month, day = map(int, date_str.split('-'))
        return cls(year, month, day)

    # 靜態方法（不接收 cls 或 self）
    @staticmethod
    def is_valid_date(year, month, day):
        """檢查日期是否有效"""
        if month < 1 or month > 12:
            return False
        if day < 1 or day > 31:
            return False
        return True

# 使用
date1 = Date(2024, 1, 15)
print(date1.display())          # 2024/01/15

date2 = Date.from_string("2024-03-20")
print(date2.display())          # 2024/03/20

print(Date.is_valid_date(2024, 2, 30))  # False
```

## 重點整理

| 概念 | 說明 | 實作方式 |
|------|------|---------|
| 類別 | 物件的藍圖 | `class ClassName:` |
| 封裝 | 隱藏內部實作 | 使用 `_` 和 `@property` |
| 繼承 | 程式碼重用 | `class Child(Parent):` |
| 多型 | 相同介面不同實作 | 方法覆寫 |
| 魔術方法 | 與 Python 整合 | `__method__()` |
| 類別方法 | 操作類別本身 | `@classmethod` |
| 靜態方法 | 工具函式 | `@staticmethod` |

## 🏆 經典練習題

### 類別與實例 🟢 簡易 | 🟡 中等

1. **學生類別**：建立一個 `Student` 類別，有 `name`、`age`、`score` 屬性，以及 `introduce()` 方法
   ```python
   s = Student("Alice", 20, 95)
   s.introduce()  # 我是 Alice，今年 20 歲，成績 95 分
   ```
   💡 提示：`__init__` 初始化屬性

2. **銀行帳戶**：實作 `BankAccount` 類別，支援 `deposit(amount)`、`withdraw(amount)`、`balance` 屬性
   💡 提示：初始餘額為 0

3. **計數器**：建立一個 `Counter` 類別，每呼叫一次 `increment()` 就 +1，`reset()` 歸零
   💡 提示：用實例變數儲存當前值

### 繼承與多型 🟡 中等 | 🔴 困難

4. **動物叫聲**：建立 `Animal` 父類別（`speak()` 方法），以及 `Dog`、`Cat`、`Duck` 子類別各自覆寫
   ```python
    for animal in [Dog(), Cat(), Duck()]:
        animal.speak()  # 汪汪！ / 喵喵！ / 呱呱！
   ```
   💡 提示：子類別用 `def speak(self):` 覆寫父類別方法

5. **幾何圖形**：建立 `Shape` 抽象類別（`area()` 方法），`Circle` 和 `Rectangle` 實作不同計算方式
   💡 提示：用 `from abc import ABC, abstractmethod`

6. **super() 應用**：建立 `Employee`（基本薪資）和 `Manager`（加給津貼），經理呼叫 `super().__init__()` 後再設定額外屬性
   💡 提示：`super().__init__(name, salary)`

### 封裝與 property 🟡 中等

7. **溫度類別**：建立 `Temperature` 類別，內部存攝氏，但提供 `celsius` 和 `fahrenheit` 兩個 property
   ```python
   t = Temperature(25)
   print(t.fahrenheit)  # 77.0
   t.fahrenheit = 100
   print(t.celsius)     # 37.78
   ```
   💡 提示：`@property` 和 `@fahrenheit.setter`

8. **密碼安全**：建立 `User` 類別，`password` 屬性用 property 保護，只能設定不能讀取
   💡 提示：getter 回傳 `"******"` 或 raise 錯誤

9. **唯讀屬性**：建立 `Book` 類別，`isbn` 在初始化後不能被修改
   💡 提示：property 只定義 getter 不定義 setter

### 特殊方法（__str__/__repr__ 等） 🟡 中等

10. **字串表示**：為 `Student` 加上 `__str__` 和 `__repr__`，讓 `print(student)` 顯示易讀內容
    💡 提示：`__str__` for 使用者，`__repr__` for 開發者

11. **向量運算**：建立 `Vector` 類別，支援 `+`（`__add__`）、`*`（`__mul__` 純量乘法）、`abs()`（`__abs__`）
    ```python
    v1 = Vector(1, 2)
    v2 = Vector(3, 4)
    print(v1 + v2)  # Vector(4, 6)
    ```
    💡 提示：魔術方法回傳新 Vector 實例

12. **可呼叫物件**：建立 `Multiplier` 類別，實作 `__call__` 讓實例可以像函式一樣被呼叫
    ```python
    double = Multiplier(2)
    print(double(5))  # 10
    ```
    💡 提示：`def __call__(self, x):`

### 類別方法與靜態方法 🟡 中等

13. **類別建構子**：為 `Date` 類別新增 `from_string(date_str)` 類別方法，解析 `"2024-01-15"` 格式
    💡 提示：`@classmethod` 第一個參數是 `cls`

14. **靜態工具方法**：在 `MathUtils` 類別中用 `@staticmethod` 定義 `is_even(n)`，不需要建立實例就能呼叫
    💡 提示：靜態方法沒有 `self` 或 `cls`

### 多重繼承與 MRO 🔴 困難

15. **鑽石問題**：建立 A → B, C → D 的鑽石繼承結構，觀察 MRO（Method Resolution Order）
    ```python
    class A: pass
    class B(A): pass
    class C(A): pass
    class D(B, C): pass
    print(D.__mro__)
    ```
    💡 提示：用 `__mro__` 或 `mro()` 查看

16. **Mixin 設計**：建立 `JSONMixin` 提供 `to_json()` 方法，讓其他類別透過多重繼承獲得此功能
    💡 提示：Mixin 不單獨使用，總是搭配其他類別

### 抽象類別 🔴 困難

17. **支付系統**：建立抽象類別 `Payment`，子類別 `CreditCardPayment`、`LinePayPayment` 各自實作 `pay(amount)`
    💡 提示：`@abstractmethod` 強制子類別實作

18. **儲存介面**：建立抽象類別 `Storage`，定義 `save(data)` 和 `load()`，分別用 `FileStorage` 和 `DBStorage` 實作
    💡 提示：相同的介面，不同的實作方式

### 設計模式入門 🔴 困難

19. **單例模式（Singleton）**：確保 `Logger` 類別只有一個實例
    ```python
    log1 = Logger()
    log2 = Logger()
    print(log1 is log2)  # True
    ```
    💡 提示：用 `__new__` 或類別變數控制實例化

20. **工廠模式**：建立 `ShapeFactory`，根據傳入的型別字串回傳對應的圖形物件
    💡 提示：`ShapeFactory.create("circle", radius=5)`

21. **觀察者模式**：實作簡化版，`NewsPublisher` 發布消息時通知所有已註冊的 `Subscriber`
    💡 提示：用列表儲存所有觀察者，發布時逐一呼叫

22. **迭代器模式**：實作一個 `Playlist` 類別，支援 `for song in playlist:` 的遍歷
    💡 提示：實作 `__iter__` 和 `__next__`
