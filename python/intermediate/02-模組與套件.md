# 02 - 模組與套件

> 🎯 **學習目標**：學習如何組織 Python 程式碼、使用模組與套件，以及管理第三方依賴。

## 什麼是模組？

模組（Module）就是一個 **`.py` 檔案**，裡面包含函式、類別和變數的定義。使用模組可以讓程式碼更容易組織和維護。

## 匯入模組

### import 語句

```python
# 匯入整個模組
import math
print(math.sqrt(16))   # 4.0
print(math.pi)         # 3.14159...

# 使用別名
import datetime as dt
today = dt.date.today()
print(today)

# 只匯入特定內容
from math import sqrt, pi
print(sqrt(25))   # 5.0（直接使用，不用 math.）

# 匯入所有內容（不建議）
from math import *  # 可能造成命名衝突
```

### 模組搜尋路徑

Python 會在以下路徑尋找模組：

```python
import sys

# 查看模組搜尋路徑
for path in sys.path:
    print(path)

# 執行結果範例：
# /usr/lib/python3.12
# /usr/lib/python3.12/site-packages
# （目前目錄）
```

## 建立自己的模組

建立一個 `my_module.py` 檔案：

```python
# my_module.py
"""這是我自己的模組"""

def greet(name):
    return f"Hello, {name}!"

PI = 3.14159

class Calculator:
    @staticmethod
    def add(a, b):
        return a + b

if __name__ == "__main__":
    # 只在直接執行此檔案時執行
    print(greet("World"))
```

在同目錄的其他檔案中使用：

```python
# main.py
import my_module

print(my_module.greet("Alice"))         # Hello, Alice!
print(my_module.PI)                     # 3.14159
calc = my_module.Calculator()
print(calc.add(5, 3))                   # 8
```

### `if __name__ == "__main__"` 的用途

這個條件式確保：
- 當模組**直接執行**時，條件為 `True`，執行測試程式碼
- 當模組**被匯入**時，條件為 `False`，不執行程式碼

## 套件（Package）

套件是包含多個模組的**目錄**，必須包含 `__init__.py` 檔案。

### 套件結構

```
my_package/
├── __init__.py      # 套件初始化（可以是空檔案）
├── module_a.py
├── module_b.py
└── sub_package/
    ├── __init__.py
    └── module_c.py
```

### `__init__.py`

```python
# my_package/__init__.py
"""My Package - 這是我建立的套件"""

# 控制對外匯出
__all__ = ["module_a", "module_b"]

# 匯入時自動執行的程式碼
print("初始化 my_package...")

# 方便使用者匯入
from .module_a import useful_function
from .module_b import HelperClass
```

### 匯入套件

```python
# 幾種匯入方式
import my_package
from my_package import module_a
from my_package.module_a import useful_function
from my_package.sub_package import module_c
```

## 第三方套件管理

### PyPI 與 pip

[PyPI](https://pypi.org/)（Python Package Index）是 Python 的官方套件倉庫。

```bash
# 安裝套件
pip install requests
pip install requests==2.31.0    # 指定版本
pip install "requests>=2.0"     # 最低版本

# 更新套件
pip install --upgrade requests

# 移除套件
pip uninstall requests

# 列出套件
pip list

# 查看套件資訊
pip show requests
```

### requirements.txt

```txt
# requirements.txt
requests==2.31.0
flask==3.0.0
numpy>=1.24.0
pandas
```

```bash
# 安裝所有套件
pip install -r requirements.txt

# 匯出目前環境的套件
pip freeze > requirements.txt
```

## 常用標準函式庫

### os - 作業系統介面

```python
import os

# 取得目前目錄
print(os.getcwd())

# 列出目錄內容
print(os.listdir("."))

# 建立目錄
os.makedirs("new_dir/sub_dir", exist_ok=True)

# 環境變數
print(os.environ.get("HOME"))
```

### sys - 系統相關

```python
import sys

# 命令列參數
print(sys.argv)  # ['script.py', 'arg1', 'arg2']

# 離開程式
sys.exit(0)      # 正常結束

# Python 版本
print(sys.version)

# 模組搜尋路徑
print(sys.path)
```

### json - JSON 處理

```python
import json

# Python 物件 → JSON 字串
data = {
    "name": "Alice",
    "age": 25,
    "skills": ["Python", "JavaScript"]
}

json_str = json.dumps(data, ensure_ascii=False, indent=2)
print(json_str)

# JSON 字串 → Python 物件
parsed = json.loads(json_str)
print(parsed["name"])  # Alice

# 讀寫 JSON 檔案
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

with open("data.json", "r", encoding="utf-8") as f:
    loaded = json.load(f)
```

### datetime - 日期時間

```python
from datetime import datetime, date, timedelta

# 現在時間
now = datetime.now()
print(now)

# 格式化
print(now.strftime("%Y-%m-%d %H:%M:%S"))
print(now.strftime("%A"))  # 星期幾

# 字串轉日期
date_str = "2024-01-15"
parsed_date = datetime.strptime(date_str, "%Y-%m-%d")

# 日期運算
tomorrow = date.today() + timedelta(days=1)
last_week = date.today() - timedelta(weeks=1)
```

### random - 隨機功能

```python
import random

# 基本功能
print(random.random())        # 0.0 ~ 1.0 之間的隨機數
print(random.randint(1, 10))  # 1 ~ 10 之間的隨機整數
print(random.uniform(1, 10)) # 1.0 ~ 10.0 之間的隨機浮點數

# 從序列中選擇
fruits = ["蘋果", "香蕉", "橘子"]
print(random.choice(fruits))       # 隨機選擇一個
print(random.sample(fruits, 2))    # 隨機選擇兩個（不重複）
random.shuffle(fruits)             # 隨機打亂順序
```

## 🏆 經典練習題

### 模組導入方式 🟢 簡易 | 🟡 中等

1. **三種導入**：分別用 `import`、`from...import`、`from...import *` 三種方式導入 `math` 模組的 `sqrt` 函式
   💡 提示：第三種方式不建議在正式專案中使用

2. **別名導入**：用 `as` 為 `numpy` 取別名 `np`，並建立一個陣列
   ```python
   import numpy as np
   arr = np.array([1, 2, 3])
   ```
   💡 提示：別名讓程式碼更簡潔

3. **尋找模組**：當你 `import something` 時，Python 去哪裡找這個模組？
   💡 提示：`sys.path` 列出了所有搜尋路徑

### 自訂套件結構 🟡 中等 | 🔴 困難

4. **建立套件**：建立一個名為 `utils` 的套件，包含 `string_utils.py` 和 `math_utils.py` 兩個模組
   ```python
   # 目錄結構：
   # utils/
   #   __init__.py
   #   string_utils.py
   #   math_utils.py
   ```
   💡 提示：`__init__.py` 可以是空檔案

5. **相對導入**：在套件內部，用相對導入引入兄弟模組
   💡 提示：`from . import sibling_module`

6. **__init__.py 活用**：在 `__init__.py` 中定義 `__all__`，控制 `from package import *` 會導入哪些名稱
   💡 提示：`__all__ = ["func_a", "ClassB"]`

### pip 與第三方套件 🟢 簡易 | 🟡 中等

7. **安裝套件**：用 pip 安裝 `requests`，並寫一支程式用 GET 請求取得 https://api.github.com 的資料
   💡 提示：`requests.get(url).json()`

8. **requirements.txt**：匯出目前環境的所有套件到檔案，然後在另一個環境安裝
   ```bash
   pip freeze > requirements.txt
   pip install -r requirements.txt
   ```
   💡 提示：這確保兩個環境的依賴一致

9. **版本指定**：安裝特定版本的 pandas（例如 2.0.0）
   💡 提示：`pip install pandas==2.0.0`

### __name__ == '__main__' 🟡 中等

10. **腳本 vs 模組**：寫一個檔案，當直接執行時印出「直接執行」，當被 import 時印出「被導入」
    💡 提示：`if __name__ == "__main__":`

11. **測試程式碼**：在模組底部用 `if __name__ == "__main__":` 區塊放測試程式碼
    💡 提示：這讓模組既可被導入，也可獨立執行測試

### 套件發布概念 🔴 困難

12. **setup.py 基礎**：寫一個簡易的 `setup.py`，包含套件名稱、版本、作者、描述
    ```python
    from setuptools import setup, find_packages
    setup(
        name="my-package",
        version="0.1.0",
        packages=find_packages()
    )
    ```
    💡 提示：`find_packages()` 自動尋找套件目錄

13. **pyproject.toml**：Python 社群已逐漸轉向 `pyproject.toml`，它和 `setup.py` 有什麼不同？
    💡 提示：`pyproject.toml` 是 PEP 518/621 制定的現代標準

14. **虛擬環境管理**：解釋為什麼每個專案都應該使用獨立的虛擬環境，並說明如何用 `venv` 建立
    💡 提示：避免套件版本衝突，`python -m venv .venv`
