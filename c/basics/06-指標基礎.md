# 06 - 指標基礎

> 🎯 **學習目標**：理解指標的核心概念——記憶體位址，學會使用指標進行間接存取，並掌握陣列與指標之間的密切關係。

---

## 什麼是記憶體位址？

電腦的記憶體就像一排編號的郵政信箱——每個箱子都有一個**唯一的號碼**（位址），裡面可以存放資料：

```
記憶體示意圖（每個格子 = 1 byte）：

位址：  0x1000  0x1001  0x1002  0x1003  0x1004  0x1005
       ┌──────┬──────┬──────┬──────┬──────┬──────┐
內容： │  42  │      │      │      │      │      │
       └──────┴──────┴──────┴──────┴──────┴──────┘
        ↑
       int x = 42; 存放在這裡（假設 int 佔 4 bytes）
```

### 取址運算子 &

在 C 語言中，你可以用 `&` 運算子取得變數的記憶體位址：

```c
#include <stdio.h>

int main() {
    int x = 42;

    printf("x 的值（內容）：%d\n", x);
    printf("x 的位址：%p\n", (void*)&x);  // %p 用來輸出位址

    return 0;
}
```

> 💡 **提示**：每次執行時，變數的位址都可能不同（因為 ASLR——位址空間配置隨機化）。但重要的是理解**變數與位址的對應關係**。

## 指標的宣告與使用

**指標（Pointer）** 是一個**存放記憶體位址**的變數。可以想像成「指向某個箱子的一張標籤」。

### 宣告指標

```c
int *p;    // p 是一個指向 int 的指標
char *cp;  // cp 是一個指向 char 的指標
double *dp; // dp 是一個指向 double 的指標
```

> 💡 **提示**：`int *p;` 可以讀作「`*p` 的型別是 `int`」，或者「`p` 是指向 `int` 的指標」。

### 取址與解參考

```c
#include <stdio.h>

int main() {
    int x = 42;
    int *p;        // 宣告一個指向 int 的指標

    p = &x;        // p 存放 x 的位址（p 指向 x）

    printf("x 的值：%d\n", x);
    printf("x 的位址（&x）：%p\n", (void*)&x);
    printf("p 的值（就是 x 的位址）：%p\n", (void*)p);
    printf("p 自己的位址：%p\n", (void*)&p);

    // 解參考（dereference）：透過 p 存取 x 的值
    printf("透過 *p 取得 x 的值：%d\n", *p);

    // 透過指標修改 x
    *p = 100;      // 等於 x = 100
    printf("修改後 x 的值：%d\n", x);

    return 0;
}
```

### 記憶體示意圖

```
int x = 42;
int *p = &x;

   p (在 0x2000)          x (在 0x1000)
┌──────────────┐      ┌──────────────┐
│   0x1000     │ ───→ │     42       │
└──────────────┘      └──────────────┘

*p 就是 x，所以 *p = 100 等同於 x = 100
```

### 宣告中的 `*` 與解參考的 `*` 是不同意思

這是初學者最常混淆的地方：

```c
int *p;    // 宣告：* 表示「p 是一個指標」
*p = 10;   // 使用：* 表示「解參考，存取 p 指向的內容」

// 宣告多個指標
int *p1, *p2;    // p1 和 p2 都是指標 ✅
int* p3, p4;     // p3 是指標，但 p4 只是 int ❌（容易混淆）

// 建議寫法：* 貼近變數名稱
int *p5, *p6;    // 清楚表示 p5、p6 都是指標
```

## NULL 指標

未指向任何有效位址的指標應設定為 `NULL`：

```c
#include <stdio.h>

int main() {
    int *p = NULL;    // p 目前不指向任何地方

    // 使用前檢查是否為 NULL
    if (p != NULL) {
        printf("*p = %d\n", *p);
    } else {
        printf("p 是 NULL，不能解參考！\n");
    }

    // ❌ 危險：對 NULL 解參考會造成 segmentation fault（程式當機）
    // *p = 100;  // 如果執行這行，程式會崩潰！

    return 0;
}
```

> ⚠️ **注意**：**永遠不要在沒有確認指標非 `NULL` 的情況下解參考指標！** Segmentation fault（段錯誤）是 C 程式中最常見的當機原因。

## 指標與函式參數：模擬傳參考呼叫

第五章說過 C 語言是傳值呼叫，函式無法修改原變數。指標讓我們可以**間接**修改原變數：

### 經典範例：swap 函式

```c
#include <stdio.h>

// ❌ 錯誤：傳值呼叫，無法交換
void swap_wrong(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}

// ✅ 正確：使用指標，間接修改原變數
void swap(int *a, int *b) {
    int temp = *a;   // 取得 a 指向的值
    *a = *b;         // 把 b 指向的值寫入 a 指向的位置
    *b = temp;       // 把暫存值寫入 b 指向的位置
}

int main() {
    int x = 10, y = 20;

    printf("交換前：x = %d, y = %d\n", x, y);

    swap_wrong(x, y);
    printf("嘗試 swap_wrong：x = %d, y = %d\n", x, y);  // 沒變！

    swap(&x, &y);
    printf("嘗試 swap（用指標）：x = %d, y = %d\n", x, y);  // 交換成功！

    return 0;
}
```

### swap 的記憶體操作示意圖

```
swap(&x, &y) 呼叫時：

main 中：           swap 中：
x 在位址 0x100     a = &x（指向 0x100）
┌──────┐            ┌──────────┐
│  10  │ ←───────── │  0x100   │  ← *a = 10
└──────┘            └──────────┘
y 在位址 0x200     b = &y（指向 0x200）
┌──────┐            ┌──────────┐
│  20  │ ←───────── │  0x200   │  ← *b = 20
└──────┘            └──────────┘

執行 *a = *b; *b = temp; 之後：

x: ┌──────┐          y: ┌──────┐
   │  20  │             │  10  │
   └──────┘             └──────┘
   交換成功！
```

### 其他使用指標參數的範例

```c
#include <stdio.h>

// 透過指標回傳多個值
void split_parts(int total, int *quotient, int *remainder) {
    *quotient = total / 3;
    *remainder = total % 3;
}

// 透過指標「更新」變數
void add_tax(double *price, double tax_rate) {
    *price = *price * (1 + tax_rate);
}

int main() {
    // 一次取得兩個回傳值
    int q, r;
    split_parts(10, &q, &r);
    printf("10 ÷ 3 = %d 餘 %d\n", q, r);

    // 更新變數
    double my_price = 100.0;
    add_tax(&my_price, 0.05);   // 加 5% 稅
    printf("含稅價格：%.2f\n", my_price);

    return 0;
}
```

## 指標算術（Pointer Arithmetic）

指標可以進行加減運算，但**意義不同於一般整數加減**：

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int *p = arr;       // p 指向 arr[0]

    printf("p    = %p, *p    = %d\n", (void*)p, *p);
    printf("p+1  = %p, *(p+1)= %d\n", (void*)(p+1), *(p+1));
    printf("p+2  = %p, *(p+2)= %d\n", (void*)(p+2), *(p+2));

    // 注意：p+1 在記憶體位址上加的是 sizeof(int) = 4 bytes！
    // 而不是加 1 byte

    return 0;
}
```

### 指標加減的意義

```
int arr[] = {10, 20, 30, 40, 50};
int *p = arr;

位址：  0x1000  0x1004  0x1008  0x100C  0x1010
       ┌──────┬──────┬──────┬──────┬──────┐
內容： │  10  │  20  │  30  │  40  │  50  │
       └──────┴──────┴──────┴──────┴──────┘
         ↑       ↑       ↑       ↑       ↑
         p      p+1     p+2     p+3     p+4
        arr[0]  arr[1]  arr[2]  arr[3]  arr[4]

*p = 10
*(p+1) = 20
*(p+2) = 30
p[2] = 30   ← 注意：p[2] 等同於 *(p+2)！
```

> 💡 **提示**：`p + n` 在位址上的增量是 `n × sizeof(指向的型別)`。這就是為什麼指標需要知道指向的型別。

### 用指標走訪陣列

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int n = sizeof(arr) / sizeof(arr[0]);

    // 方式一：用陣列索引
    printf("陣列索引寫法：");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // 方式二：用指標
    printf("指標寫法：");
    for (int *p = arr; p < arr + n; p++) {
        printf("%d ", *p);
    }
    printf("\n");

    return 0;
}
```

## 陣列與指標的關係

在 C 語言中，**陣列名稱本身就是一個指標常數**——指向陣列的第一個元素：

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};

    // 陣列名稱就是第一個元素的位址
    printf("arr  = %p\n", (void*)arr);
    printf("&arr[0] = %p\n", (void*)&arr[0]);

    // 陣列名稱已經是一個位址，不需要 &
    int *p = arr;          // 不需要 &arr
    // int *p = &arr[0];   // 這也等價

    // arr[i] 等價於 *(arr + i)
    printf("arr[2] = %d\n", arr[2]);        // 30
    printf("*(arr+2) = %d\n", *(arr + 2));  // 30

    // 但陣列名稱不是變數——不能修改
    // arr++;     // ❌ 錯誤：陣列名稱是常數
    p++;           // ✅ 指標可以修改

    return 0;
}
```

### 方括號的本質

```c
arr[i]   →   *(arr + i)    // 陣列寫法
i[arr]   →   *(i + arr)    // 對稱性——這在 C 語言中真的合法！

// 範例：
int arr[5] = {10, 20, 30, 40, 50};
printf("%d\n", 2[arr]);     // 輸出 30（等於 arr[2]）
```

> 💡 **提示**：雖然 `2[arr]` 語法上正確，但**不要這樣寫**——它會讓閱讀程式碼的人困惑。

### 陣列與指標的差異總結

| 特性 | 陣列 | 指標 |
|------|------|------|
| 儲存方式 | 儲存實際資料 | 儲存位址 |
| 可否修改 | ❌ 陣列名稱不可改 | ✅ 可以指向其他地方 |
| `sizeof` | 整個陣列大小 | 指標本身的大小（4 or 8 bytes） |
| `&` | `&arr` 與 `arr` 相同 | `&p` 是指標自己的位址 |

```c
#include <stdio.h>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int *p = arr;

    printf("sizeof(arr) = %zu\n", sizeof(arr));  // 20（5 × 4）
    printf("sizeof(p)  = %zu\n", sizeof(p));     // 8（64 位元系統）

    return 0;
}
```

## void* 泛型指標

`void*` 是不指定型別的指標，可以指向任何型別，但在解參考前**必須先轉型**：

```c
#include <stdio.h>

int main() {
    int x = 42;
    double y = 3.14;

    void *vp;

    vp = &x;
    printf("int 值：%d\n", *(int*)vp);     // 先轉 int* 再解參考

    vp = &y;
    printf("double 值：%lf\n", *(double*)vp);  // 先轉 double* 再解參考

    return 0;
}
```

> 💡 **提示**：`void*` 常用於泛型函式的實作（如 `qsort`、`memcpy`），一般程式開發較少直接使用。

## 常見指標錯誤

### 1️⃣ 未初始化指標

```c
int *p;        // p 指向哪裡？不知道！
*p = 42;       // ❌ 不確定會寫入哪個記憶體位置！
```

**解決**：宣告時就初始化，指向已知位置或設為 NULL。

### 2️⃣ 對 NULL 解參考

```c
int *p = NULL;
*p = 42;       // ❌ Segmentation fault！
```

**解決**：使用前檢查 `if (p != NULL)`。

### 3️⃣ 懸吊指標（Dangling Pointer）

```c
int *p;
{
    int x = 42;
    p = &x;
}
// 離開區塊後，x 已被釋放
*p = 100;      // ❌ 懸吊指標！不確定行為
```

**解決**：確保指標指向的記憶體仍然有效。

### 4️⃣ 指標型別不匹配

```c
int x = 42;
double *p = &x;      // ❌ 型別不匹配
*p = 3.14;           // 可能破壞其他記憶體！
```

**解決**：確保指標型別與指向的資料型別一致。

## 本章重點整理

| 概念 | 重點 |
|------|------|
| 指標 | 存放記憶體位址的變數 |
| `&` 運算子 | 取得變數位址 |
| `*` 運算子（解參考） | 存取指標指向的內容 |
| `NULL` | 代表指標不指向任何地方 |
| 指標算術 | `p+n` 移動 `n × sizeof(型別)` 個位元組 |
| 陣列與指標 | `arr[i]` 等價於 `*(arr + i)` |
| 指標參數 | 讓函式可以修改原變數（模擬傳參考） |
| `void*` | 泛型指標，使用前需轉型 |
| 安全檢查 | 永遠確認指標不是 NULL 再解參考 |
