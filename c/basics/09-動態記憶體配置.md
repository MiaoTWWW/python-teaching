# 09 - 動態記憶體配置

> 🎯 **學習目標**：理解堆積（Heap）記憶體的概念，學會使用 malloc、calloc、realloc 動態配置記憶體，並養成正確的記憶體管理習慣。

---

## 為什麼需要動態記憶體配置？

到目前為止，我們使用的都是**靜態陣列**——大小在編譯時期就決定了：

```c
int scores[100];  // 只能存 100 個學生的成績
```

靜態陣列有兩個主要問題：

| 問題 | 說明 |
|------|------|
| **固定大小** | 如果使用者需要 101 個，程式就無法處理 |
| **浪費空間** | 如果使用者只需要 10 個，其他 90 個空間就浪費了 |
| **執行期決定** | 使用者在執行程式前，我們常常不知道需要多少空間 |

**動態記憶體配置**讓程式在**執行時期**向作業系統請求需要的記憶體大小，用完後再歸還。這讓我們可以寫出更有彈性的程式。

### 靜態 vs 動態

| 特性 | 靜態陣列 | 動態記憶體 |
|------|---------|-----------|
| 大小決定時機 | 編譯時期 | 執行時期 |
| 存放位置 | Stack（堆疊） | Heap（堆積） |
| 大小彈性 | 固定 | 可調整（realloc） |
| 生命週期 | 離開作用域自動釋放 | 手動釋放（free） |
| 效能 | 較快 | 較慢（需要系統呼叫） |

---

## 記憶體布局回顧

C 程式的記憶體在執行時期分為以下區域：

```
高記憶體位址
  ┌──────────────────┐
  │     Stack        │ ← 區域變數、函式呼叫框架（自動管理）
  │     （堆疊）      │
  ├──────────────────┤
  │      ↓↓          │
  │                  │
  │      ↑↑          │
  ├──────────────────┤
  │     Heap         │ ← malloc 配置的記憶體（手動管理）
  │     （堆積）      │
  ├──────────────────┤
  │    Data Segment  │ ← 全域變數、static 變數
  ├──────────────────┤
  │   Text Segment   │ ← 程式碼（唯讀）
  │   （程式碼）      │
  └──────────────────┘
低記憶體位址
```

> 💡 **提示**：Stack 和 Heap 是從兩端往中間生長的。Stack 由編譯器自動管理（函式呼叫時推入、返回時彈出），Heap 則需要程式設計師手動管理（malloc/free）。

## malloc — 記憶體配置

`malloc`（memory allocation）從 Heap 中配置指定大小的記憶體區塊：

```c
#include <stdlib.h>  // malloc 需要這個標頭檔

void* malloc(size_t size);
```

- 參數 `size`：要配置的位元組數
- 回傳值：配置成功的記憶體起始位址（`void*`），失敗則回傳 `NULL`
- 配置的記憶體**內容不會被初始化**（裡面是垃圾值）

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;

    printf("請輸入陣列大小：");
    scanf("%d", &n);

    // 動態配置 n 個整數的空間
    int *arr = (int*)malloc(n * sizeof(int));

    // ⚠️ 一定要檢查 malloc 是否成功！
    if (arr == NULL) {
        printf("記憶體配置失敗！\n");
        return 1;
    }

    // 使用配置的記憶體
    for (int i = 0; i < n; i++) {
        arr[i] = i * 10;
    }

    for (int i = 0; i < n; i++) {
        printf("arr[%d] = %d\n", i, arr[i]);
    }

    // 用完後釋放記憶體
    free(arr);

    return 0;
}
```

### malloc 的關鍵要點

```c
// 配置 10 個 int
int *p = (int*)malloc(10 * sizeof(int));

// ❌ 常見錯誤：忘記乘上 sizeof(int)
int *p = (int*)malloc(10);  // 只配置了 10 bytes，不是 10 個 int！

// ❌ 常見錯誤：在 32 位元系統上使用 sizeof 不當
int *p = (int*)malloc(10 * sizeof(int));  // ✅ 安全寫法

// ✅ 另一個安全寫法：不重複型別名稱
int *p = malloc(10 * sizeof(*p));  // sizeof(*p) = sizeof(int)
```

> 💡 **提示**：`malloc` 回傳的是 `void*`，在 C 語言中**不需要**強制轉型（`void*` 可隱式轉換為任何指標型別）。但許多程式設計師仍會加上轉型以增加可讀性。在 C++ 中則**必須**轉型。

### 為什麼要檢查 NULL？

```c
int *p = (int*)malloc(999999999999);  // 超大配置，可能失敗

if (p == NULL) {
    // 記憶體不足，需要處理錯誤
    fprintf(stderr, "記憶體配置失敗！\n");
    return 1;
}

// 如果 p 是 NULL 卻繼續使用：
// p[0] = 42;  // ❌ Segfault！對 NULL 指標解參考
```

> ⚠️ **注意**：**永遠檢查 malloc 的回傳值！** 雖然在一般的 PC 開發中很少遇到配置失敗，但在嵌入式系統、伺服器或處理大型資料時，記憶體不足是真實存在的風險。

## calloc — 配置並初始化為 0

`calloc`（clear allocation）與 `malloc` 類似，但會將記憶體內容全部設為 0：

```c
void* calloc(size_t num, size_t size);
```

- `num`：元素數量
- `size`：每個元素的大小（bytes）
- 回傳值：配置成功的記憶體起始位址，失敗回傳 `NULL`

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n = 5;

    // malloc：內容是垃圾值
    int *m = (int*)malloc(n * sizeof(int));
    printf("malloc：");
    for (int i = 0; i < n; i++) printf("%d ", m[i]);  // 垃圾值
    printf("\n");

    // calloc：內容全部為 0
    int *c = (int*)calloc(n, sizeof(int));
    printf("calloc：");
    for (int i = 0; i < n; i++) printf("%d ", c[i]);  // 0 0 0 0 0
    printf("\n");

    free(m);
    free(c);

    return 0;
}
```

### malloc vs calloc

| 比較 | malloc | calloc |
|------|--------|--------|
| 參數 | `size`（總大小） | `num` × `size`（數量×元素大小） |
| 初始化 | ❌ 不初始化（垃圾值） | ✅ 全部設為 0 |
| 效能 | 較快（不須寫入 0） | 較慢（需要寫入 0） |
| 使用時機 | 馬上會初始化所有元素 | 需要預設為 0 |

> 💡 **提示**：如果你在 malloc 後馬上用 `for` 迴圈設定所有元素，使用 `calloc` 可能更簡潔。但如果你只使用部分的元素，`malloc` 較快。

## realloc — 重新調整大小

`realloc`（re-allocation）可以調整已配置記憶體區塊的大小——可能擴大或縮小：

```c
void* realloc(void *ptr, size_t new_size);
```

- `ptr`：先前由 malloc/calloc/realloc 回傳的指標
- `new_size`：新的總大小（bytes）
- 回傳值：調整大小後的記憶體位址（可能與原位址不同），失敗回傳 NULL

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    // 先配置 5 個 int
    int *arr = (int*)malloc(5 * sizeof(int));
    if (arr == NULL) return 1;

    for (int i = 0; i < 5; i++) arr[i] = i;

    printf("原陣列：");
    for (int i = 0; i < 5; i++) printf("%d ", arr[i]);
    printf("\n");

    // 擴大到 10 個 int
    int *temp = (int*)realloc(arr, 10 * sizeof(int));
    if (temp == NULL) {
        printf("realloc 失敗，原資料仍在 arr 中\n");
        free(arr);
        return 1;
    }
    arr = temp;  // 更新指標

    // 新擴展的部分是垃圾值，需要初始化
    for (int i = 5; i < 10; i++) arr[i] = i * 10;

    printf("擴展後：");
    for (int i = 0; i < 10; i++) printf("%d ", arr[i]);
    printf("\n");

    free(arr);
    return 0;
}
```

### ⚠️ realloc 的重要注意事項

```c
// ❌ 錯誤：直接使用 realloc 的回傳值覆蓋原指標
int *arr = (int*)malloc(10 * sizeof(int));
arr = (int*)realloc(arr, 100 * sizeof(int));  // 如果 realloc 失敗，arr 變成 NULL
// 原記憶體也無法存取了（記憶體洩漏！）
```

```c
// ✅ 正確：使用暫存指標
int *arr = (int*)malloc(10 * sizeof(int));
int *temp = (int*)realloc(arr, 100 * sizeof(int));
if (temp != NULL) {
    arr = temp;  // realloc 成功，更新指標
} else {
    // realloc 失敗，arr 仍然指向原記憶體
    printf("realloc 失敗\n");
    // 仍可以使用 arr，記得要 free
}
```

### realloc 的幾種特殊行為

| 情況 | 行為 |
|------|------|
| `ptr` 為 NULL | 等同於 `malloc(new_size)` |
| `new_size` 為 0 | 實作定義（通常等同於 free，但不建議這樣用） |
| 擴大 | 新增加的位元組內容未初始化 |
| 無法在原位置擴大 | 複製資料到新的位置，自動 free 原記憶體 |

```c
// realloc(NULL, size) 等同於 malloc(size)
int *p = (int*)realloc(NULL, 10 * sizeof(int));  // 等同 malloc
free(p);
```

## free — 釋放記憶體

動態配置的記憶體用完後，必須用 `free` 歸還給系統：

```c
void free(void *ptr);
```

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = (int*)malloc(5 * sizeof(int));
    // ... 使用 p ...

    free(p);      // 釋放記憶體
    p = NULL;     // ✅ 好習慣：釋放後設為 NULL
    return 0;
}
```

### free 的常見錯誤

#### 1️⃣ 忘記 free（記憶體洩漏）

```c
void leak() {
    int *p = (int*)malloc(100 * sizeof(int));
    // 忘記 free(p)！
    // 函式結束後，無法再存取這塊記憶體，但系統仍認為它被佔用
}
```

#### 2️⃣ 雙重釋放（Double Free）

```c
int *p = (int*)malloc(10 * sizeof(int));
free(p);
free(p);  // ❌ 未定義行為！釋放已釋放的記憶體
```

#### 3️⃣ 釋放後繼續使用（Use After Free）

```c
int *p = (int*)malloc(10 * sizeof(int));
free(p);
p[0] = 42;  // ❌ 未定義行為！記憶體已歸還
```

✅ **最佳實踐**：

```c
int *p = (int*)malloc(10 * sizeof(int));
// ... 使用 ...
free(p);
p = NULL;        // 設為 NULL，如果不小心再次 free(p) 不會出錯
// free(NULL);   // ✅ 這是安全的，C 標準保證 free(NULL) 不做任何事
```

### 誰該負責 free？

這是一個重要的設計問題。規則是：**誰配置（malloc），誰釋放（free）**。

```c
// 好的設計：配置與釋放在同一層級
void example() {
    int *data = (int*)malloc(100 * sizeof(int));
    // ... 使用 data ...
    free(data);
}

// 需要約定的設計：文件說清楚誰負責釋放
char* read_file(const char *filename) {
    // 函式內部 malloc
    char *content = (char*)malloc(1000);
    // ... 讀取檔案 ...
    return content;  // 呼叫者必須 free 這個回傳值
}

int main() {
    char *text = read_file("test.txt");
    // ... 使用 text ...
    free(text);  // 呼叫者負責釋放
    return 0;
}
```

## 記憶體洩漏（Memory Leak）

### 什麼是記憶體洩漏？

當你 `malloc` 了一塊記憶體，用完後卻忘了 `free`，這塊記憶體在程式結束前都無法再被使用——這就是**記憶體洩漏**。

```c
#include <stdio.h>
#include <stdlib.h>

void memory_leak_example() {
    for (int i = 0; i < 1000; i++) {
        int *p = (int*)malloc(1000 * sizeof(int));
        // 每次迴圈都配置新的記憶體，但從不釋放！
        // p 離開作用域後，這塊記憶體就再也無法存取了
    }
    // 程式已經洩漏了 1000 × 4000 ≈ 4MB 的記憶體
}
```

### 洩漏的後果

- 程式的記憶體使用量持續增長
- 最終導致系統記憶體不足（OOM，Out Of Memory）
- 作業系統會殺掉程式（OOM Killer）
- 伺服器程式長時間運行會因記憶體洩漏而崩潰

### 如何避免記憶體洩漏？

| 策略 | 說明 |
|------|------|
| **配對管理** | 每個 malloc 都有對應的 free，在寫 malloc 的同時就寫好 free |
| **明確所有權** | 文件說清楚誰負責釋放記憶體 |
| **提早釋放** | 用完了就 free，不要等到函式結尾 |
| **使用工具檢測** | Valgrind、AddressSanitizer、LeakSanitizer |

### 使用 Valgrind 檢測記憶體洩漏（Linux）

```bash
# 編譯時加上 -g 產生除錯資訊
gcc -g program.c -o program

# 用 Valgrind 執行程式
valgrind --leak-check=full ./program
```

Valgrind 的輸出範例：

```
==12345== 400 bytes in 1 blocks are definitely lost in loss record 1 of 1
==12345==    at 0x4848899: malloc (in /usr/lib/valgrind/...)
==12345==    by 0x1091B5: memory_leak_example (program.c:8)
```

### 使用 AddressSanitizer（GCC/Clang）

```bash
# 編譯時加上 -fsanitize=address
gcc -g -fsanitize=address program.c -o program
# 執行程式時會自動檢查記憶體錯誤
./program
```

## 動態二維陣列

使用指標的指標（`int **`）可以建立動態的二維陣列：

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int rows = 3, cols = 4;

    // 第一層：配置 rows 個指標
    int **matrix = (int**)malloc(rows * sizeof(int*));
    if (matrix == NULL) return 1;

    // 第二層：為每一列配置 cols 個 int
    for (int i = 0; i < rows; i++) {
        matrix[i] = (int*)malloc(cols * sizeof(int));
        if (matrix[i] == NULL) {
            // 如果某一列配置失敗，需要釋放已配置的記憶體
            for (int j = 0; j < i; j++) free(matrix[j]);
            free(matrix);
            return 1;
        }
    }

    // 填入資料
    int count = 1;
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < cols; j++) {
            matrix[i][j] = count++;
        }
    }

    // 輸出
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < cols; j++) {
            printf("%3d ", matrix[i][j]);
        }
        printf("\n");
    }

    // 釋放：反向順序，先釋放每一列，再釋放第一層
    for (int i = 0; i < rows; i++) {
        free(matrix[i]);
    }
    free(matrix);

    return 0;
}
```

> 💡 **提示**：與靜態二維陣列不同，動態二維陣列的每一列可以不連續地存在記憶體中，且每一列的長度可以不同（參差不齊的陣列，jagged array）。

## 常見的記憶體錯誤

| 錯誤類型 | 範例 | 後果 | 解決方法 |
|---------|------|------|---------|
| 忘記 free | `malloc` 後不 `free` | 記憶體洩漏 | 配對管理 |
| 雙重釋放 | 兩次 `free(p)` | 當機或資料損毀 | free 後設 NULL |
| 使用已釋放記憶體 | `free(p)` 後 `*p = 0` | 未定義行為 | free 後不再使用 |
| 緩衝區溢位 | `p[100]` 但只配置 50 | 破壞其他資料 | 嚴格檢查邊界 |
| 對 NULL 解參考 | 未檢查 malloc 回傳值 | Segfault | 每次 malloc 後檢查 |
| 使用未初始化的 malloc 記憶體 | 讀取未賦值的元素 | 不確定的值 | 使用 calloc 或初始化 |

## 本章重點整理

| 概念 | 重點 |
|------|------|
| Heap（堆積） | 動態記憶體存放的區域，需手動管理 |
| `malloc` | 配置記憶體（不初始化），回傳 `void*` |
| `calloc` | 配置記憶體並初始化為 0 |
| `realloc` | 調整已配置記憶體的大小 |
| `free` | 釋放動態配置的記憶體 |
| 檢查 NULL | 每次配置後必須檢查回傳值 |
| 暫存指標 | realloc 時使用暫存指標避免記憶體洩漏 |
| 記憶體洩漏 | malloc 後忘記 free，造成記憶體浪費 |
| 記憶體檢測 | Valgrind（Linux）、AddressSanitizer（GCC/Clang） |

## 練習題

### 練習 1：動態陣列操作

撰寫一個程式：
1. 詢問使用者要輸入幾個數字
2. 用 malloc 配置對應大小的整數陣列
3. 讓使用者逐一輸入數字
4. 計算並輸出總和與平均值
5. 釋放記憶體

### 練習 2：陣列擴展

撰寫一個程式，先配置 5 個 int 的陣列並填入值，然後使用 realloc 擴展到 10 個，讓使用者輸入新的 5 個數字，最後輸出所有 10 個數字。

### 練習 3：字串複製

不使用 `strcpy`，自己實作 `char* my_strdup(const char *s)`，使用 malloc 配置足夠的空間，複製字串內容，然後回傳新字串的指標。

**提示**：用 `strlen` 取得長度，加上 1 給結尾的 `\0`。

### 練習 4：參差不齊的陣列（Jagged Array）

建立一個動態二維陣列，其中：
- 第一列有 1 個元素
- 第二列有 2 個元素
- 第三列有 3 個元素
- 以此類推，直到第 n 列

填入數字（從 1 開始遞增），然後輸出這個三角形陣列，最後釋放所有記憶體。
