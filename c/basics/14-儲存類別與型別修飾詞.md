# 14 - 儲存類別、生命週期與型別修飾詞

> 🎯 **學習目標**：理解 C 語言的變數儲存類別（storage class）、生命週期（lifetime）、連結範疇（linkage）以及特殊型別修飾詞，掌握精確控制記憶體與變數行為的能力。

---

## 變數的四個基本屬性

C 語言中，每個變數都有四個維度的屬性：

| 屬性 | 說明 | 可能值 |
|------|------|--------|
| **儲存類別** | 變數存放在哪裡 | `auto`、`register`、`static`、`extern`、`_Thread_local` |
| **生命週期** | 變數存在多久 | 自動（區塊）生命期、靜態（程式）生命期、執行緒生命期 |
| **連結範疇** | 哪些檔案可以看到它 | 無連結、內部連結、外部連結 |
| **型別修飾詞** | 存取方式與最佳化提示 | `const`、`volatile`、`restrict` |

> 💡 大多數現代語言簡化或隱藏了這些概念，但 C 讓程式設計師**完全控制**——這也是 C 能寫作業系統和嵌入式韌體的原因。

## 儲存類別（Storage Class）

### auto — 自動變數（幾乎不用寫）

「自動」指的是**在進入區塊時自動分配、離開時自動釋放**。

```c
#include <stdio.h>

int main() {
    auto int x = 10;   // 等於 int x = 10;
    // auto 關鍵字幾乎沒人在用——區域變數預設就是 auto

    printf("%d\n", x);
    return 0;
}
```

> 💡 `auto` 在 C 中只是廢棄的遺產語法。在 C++11 之後 `auto` 被賦予了全新的意義，但**在 C 中永遠不要寫 `auto`**。

### register — 建議放在 CPU 暫存器

```c
#include <stdio.h>

int main() {
    register int counter = 0;  // 建議編譯器將 counter 放在 CPU 暫存器

    for (register int i = 0; i < 1000000; i++) {
        counter += i;
    }

    // ⚠️ 無法對 register 變數取址！
    // int *p = &counter;  // ❌ 編譯錯誤

    printf("結果：%d\n", counter);
    return 0;
}
```

| 特性 | 說明 |
|------|------|
| 用途 | 暗示編譯器「這個變數使用很頻繁，建議放在暫存器」 |
| 限制 | 不能對其取址（`&` 運算子不可用） |
| 現狀 | 現代編譯器最佳化遠比程式設計師手動標 `register` 聰明，**幾乎不需要使用** |

> ⚠️ `register` 在 C++17 已被移除，C 語言保留它只是為了向後相容。不要依賴它。

### static — 三種完全不同的用途

`static` 是 C 語言中最被誤解、用途最多的關鍵字：

#### 用途 1：區域靜態變數（延長生命週期）

```c
#include <stdio.h>

int next_id() {
    static int id = 0;   // ❗ 只會在第一次呼叫時初始化一次
    return ++id;         // 離開函式後，id 不會消失
}

int main() {
    printf("%d\n", next_id());  // 1
    printf("%d\n", next_id());  // 2
    printf("%d\n", next_id());  // 3
    // 每次呼叫 id 都還記得上一次的值！
    return 0;
}
```

區域 `static` 變數的特點：

| 特性 | 說明 |
|------|------|
| 生命週期 | 程式執行期間（不是函式執行期間） |
| 初始化 | 只初始化一次（第一次到達宣告處） |
| 可視範圍 | 只在宣告它的函式/區塊內 |
| 未明確初始化 | 自動初始化為 0 |

```c
void counter() {
    static int count;    // 自動初始化為 0
    printf("第 %d 次呼叫\n", ++count);
}
```

> 💡 應用場景：函式需要記住狀態（如產生 ID、統計呼叫次數），但不希望用全域變數污染命名空間。

#### 用途 2：限制全域變數/函式的連結範疇（檔案私有）

```c
// helper.c
static int internal_count = 0;   // 只有 helper.c 看得到
static void private_helper() {   // 只有 helper.c 看得到
    internal_count++;
}

void public_api() {               // 其他檔案可以呼叫
    private_helper();
}
```

| `static`（檔案層級） | 效果 |
|----------------------|------|
| `static int x;` | x 只能在同一個 .c 檔案內存取 |
| `static void f();` | f 只能在同一個 .c 檔案內呼叫 |

> 💡 這是 C 語言**實現封裝（encapsulation）**的方式——把不該暴露的函式和變數用 `static` 隱藏起來。

#### 用途 3：靜態陣列參數（C23 新語法）

```c
// C23 之後，static 可以用在陣列參數，指定最小長度
void process(int arr[static 10]) {
    // 編譯器可以假設 arr 至少指向 10 個整數的空間
}
```

### extern — 引用其他檔案的變數

```c
// file1.c
int global_count = 0;        // 定義：實際分配記憶體
void increment() {
    global_count++;
}

// file2.c
extern int global_count;     // 宣告：告訴編譯器「這個變數在其他檔案」
void print_count() {
    printf("%d\n", global_count);
}
```

| `extern` | 是宣告還是定義？ |
|----------|----------------|
| `extern int x;` | ❗ 只是宣告（不分配記憶體），x 的定義在其他地方 |
| `int x;` | 定義（分配記憶體） |
| `extern int x = 5;` | 定義（因為有初始值） |

```c
// 常見模式：標頭檔放 extern 宣告，.c 檔放定義

// header.h
#ifndef HEADER_H
#define HEADER_H
extern int debug_level;   // 宣告（不是定義）
#endif

// header.c
int debug_level = 3;      // 定義
```

> ⚠️ **千萬不要在 .h 檔定義變數！** 如果 `int debug_level;` 放在標頭檔，被兩個 .c 檔案 include 就會造成重複定義錯誤。

### _Thread_local — 執行緒區域儲存（C11）

```c
#include <stdio.h>
#include <threads.h>

_Thread_local int tls_var = 0;  // 每個執行緒有自己的副本

int thread_func(void *arg) {
    tls_var++;    // 各執行緒互不干擾
    printf("執行緒 %ld: tls_var = %d\n", (long)arg, tls_var);
    return 0;
}

int main() {
    thrd_t t1, t2;
    thrd_create(&t1, thread_func, (void*)1);
    thrd_create(&t2, thread_func, (void*)2);
    thrd_join(t1, NULL);
    thrd_join(t2, NULL);
    return 0;
}
```

### 儲存類別組合範例

```c
static _Thread_local int per_thread_cache = 0;
// 每個執行緒各自獨立、但只在這個檔案內可見
```

## 生命週期（Lifetime）彙整

| 生命週期 | 何時分配 | 何時釋放 | 適用的變數 |
|----------|---------|---------|-----------|
| **自動（區塊）** | 進入區塊時 | 離開區塊時 | 一般區域變數（無 `static`） |
| **靜態（程式）** | 程式啟動時 | 程式結束時 | 全域變數、`static` 變數 |
| **執行緒** | 執行緒建立時 | 執行緒結束時 | `_Thread_local` 變數 |
| **動態配置** | `malloc` 呼叫時 | `free` 呼叫時 | 堆積（heap）上的變數 |

## 連結範疇（Linkage）彙整

| 連結範疇 | 說明 | 如何達成 |
|----------|------|---------|
| **無連結** | 只能在同一個區塊內存取 | 區域變數（無 `static`、無 `extern`） |
| **內部連結** | 只能在同一個檔案內存取 | 檔案層級 `static` 變數/函式 |
| **外部連結** | 所有檔案都可存取 | 檔案層級非 `static` 變數/函式、`extern` |

```c
int global;          // 外部連結（所有 .c 都可 access）
static int hidden;   // 內部連結（只有這個檔案）

void func() {
    int local;       // 無連結（只有這個函式）
    static int cnt;  // 無連結（但生命週期是靜態的）
}
```

## 型別修飾詞（Type Qualifiers）

### const — 唯讀

```c
const double PI = 3.14159;
// PI = 3.0;           // ❌ 編譯錯誤

int x = 42;
const int *p1 = &x;    // 指向 const int 的指標：不能改 *p1
int * const p2 = &x;   // const 指標：不能改 p2 本身
const int * const p3 = &x;  // 都不能改
```

| 寫法 | 不能修改的對象 |
|------|--------------|
| `const int *p` | `*p`（指向的資料） |
| `int * const p` | `p`（指標本身） |
| `const int * const p` | 兩者都不能改 |

### volatile — 告訴編譯器「不要最佳化」

```c
#include <stdio.h>

volatile int hardware_flag = 0;  // 這個值可能被硬體或中斷改變

int main() {
    while (hardware_flag == 0) {
        // 即使沒有對 hardware_flag 賦值，編譯器也不能把這個迴圈最佳化掉
    }
    printf("硬體訊號已觸發！\n");
    return 0;
}
```

使用 `volatile` 的典型場景：

| 場景 | 說明 |
|------|------|
| **記憶體映射 I/O** | 硬體暫存器的值可能隨時改變 |
| **中斷處理** | 中斷服務程式會修改全域變數 |
| **多執行緒共享變數** | 其他執行緒可能修改該變數（C11 後建議用 atomic） |

> ⚠️ `volatile` **不是**執行緒同步機制！不要用它來取代 mutex 或 atomic。`volatile` 只是防止編譯器最佳化，不保證記憶體序（memory ordering）。

### restrict — 唯一別名承諾（C99）

```c
#include <stdio.h>
#include <string.h>

// restrict 承諾：src 和 dest 指向的記憶體區域不會重疊
void my_copy(void * restrict dest, const void * restrict src, size_t n) {
    // 編譯器可以放心地最佳化，不用擔心 src 和 dest 重疊
    for (size_t i = 0; i < n; i++) {
        ((char*)dest)[i] = ((char*)src)[i];
    }
}

int main() {
    char buf[100] = "Hello, World!";
    char copy[100];

    my_copy(copy, buf, strlen(buf) + 1);
    printf("%s\n", copy);

    // ❌ 如果 src 和 dest 重疊，行為未定義
    // my_copy(buf + 1, buf, 10);  // 未定義行為！
    return 0;
}
```

為什麼需要 `restrict`？

```c
// 沒有 restrict：編譯器必須保守
void add_arrays(int *a, int *b, int *c, int n) {
    for (int i = 0; i < n; i++) {
        a[i] = b[i] + c[i];
    }
    // 編譯器無法確定 a、b、c 是否指向同一個陣列
}

// 有 restrict：編譯器可以積極最佳化
void add_arrays_fast(int * restrict a, int * restrict b, int * restrict c, int n) {
    for (int i = 0; i < n; i++) {
        a[i] = b[i] + c[i];
    }
}
```

> 💡 `restrict` 是 C99 加入的關鍵字，C++ 沒有（直到最近 C++23 才加入 `std::restrict`）。這是 C 語言讓程式設計師給編譯器**更多最佳化資訊**的典型例子。

static const volatile int * restrict ptr;

## 本章重點整理

| 概念 | 說明 |
|------|------|
| `auto` | 預設的儲存類別，區域變數專用（不必寫） |
| `register` | 建議放暫存器（現代編譯器不需要） |
| `static`（區域） | 變數生命週期延長為整個程式，只初始化一次 |
| `static`（全域） | 限制連結範疇為當前檔案 |
| `extern` | 引用其他檔案的全域變數/函式 |
| `_Thread_local` | 每個執行緒獨立的副本（C11） |
| `const` | 唯讀，不可修改 |
| `volatile` | 禁止編譯器最佳化，用於硬體/中斷情境 |
| `restrict` | 承諾沒有別名，讓編譯器最佳化（C99） |
