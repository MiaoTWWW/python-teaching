# 15 - C99/C11 現代語法特性

> 🎯 **學習目標**：學習 C99、C11、C23 等現代 C 標準加入的實用語法特性，寫出更安全、更簡潔的 C 程式碼。

---

## C 語言的版本演進

C 語言自 1972 年誕生以來，經歷了幾次重要的標準化：

| 標準 | 俗稱 | 重要新特性 |
|------|------|-----------|
| C89/C90 | ANSI C / ISO C | 第一個標準化版本 |
| **C99** | ISO C 1999 | `//` 註解、`restrict`、`inline`、可變長度陣列（VLA，C11 後改為選用）、複合字面常數、具名初始值、`stdint.h`、`inttypes.h` |
| **C11** | ISO C 2011 | 匿名結構/匿名 union、`_Static_assert`、`_Generic`、`_Alignas`、執行緒支援、無號整數溢位安全、_Noreturn |
| C17 | ISO C 2018 | 主要是缺陷修訂，沒有新功能 |
| **C23** | ISO C 2024 | `#elifdef` / `#elifndef`、`auto` 推導（類似 C++）、`nullptr`、`constexpr`、`static` 陣列參數強化、屬性語法 `[[...]]` |

> 💡 通常編譯器在新標準正式發布前就實作了大部分功能。GCC 與 Clang 對 C99/C11 的支援已經非常成熟。

## 複合字面常數（Compound Literals，C99）

不需宣告變數就能直接建立「匿名物件」：

```c
#include <stdio.h>

typedef struct {
    int x, y;
} Point;

void print_point(Point p) {
    printf("(%d, %d)\n", p.x, p.y);
}

int main() {
    // ✅ 複合字面常數：建立匿名 Point 結構
    print_point((Point){3, 7});           // 輸出：(3, 7)

    // 也可以用在陣列
    int *arr = (int[]){1, 2, 3, 4, 5};    // 建立長度為 5 的匿名陣列
    printf("%d\n", arr[2]);               // 3

    // 在運算式中直接使用
    int sum = ((int[]){10, 20, 30})[0] +  // 10
              ((int[]){10, 20, 30})[1] +  // 20
              ((int[]){10, 20, 30})[2];   // 30
    printf("sum = %d\n", sum);            // 60

    return 0;
}
```

> 💡 複合字面常數的生命週期取決於其所處的區塊——如果在函式內，到區塊結束時就會消失。如果要在函式外使用，請用 `static`：`(const int[]){1, 2, 3}`。

實際應用——不定長度引數的最後一個元素標記：

```c
int sum_until_zero(int first, ...) {
    int total = first;
    va_list args;
    va_start(args, first);
    int n;
    while ((n = va_arg(args, int)) != 0) {
        total += n;
    }
    va_end(args);
    return total;
}

// 不需要額外宣告變數來當哨兵
int s = sum_until_zero(1, 2, 3, 4, 5, (int){0});
//                               最後一個 0 用複合字面常數表達
```

## 具名初始值（Designated Initializers，C99）

不用按照結構成員或陣列元素的順序初始化：

### 陣列具名初始值

```c
#include <stdio.h>

int main() {
    // 傳統方式：需要記住每個索引的位置
    int arr1[10] = {0, 0, 0, 0, 0, 0, 0, 42, 0, 99};

    // ✅ 具名初始值：清楚標示索引
    int arr2[10] = {[7] = 42, [9] = 99};
    // 沒指定的元素自動初始化為 0

    for (int i = 0; i < 10; i++) {
        printf("%d ", arr2[i]);
    }
    // 輸出：0 0 0 0 0 0 0 42 0 99

    // 指定範圍（GCC 延伸，非標準 C，但廣泛支援）
    int arr3[100] = {[0 ... 49] = 1, [50 ... 99] = 2};

    return 0;
}
```

### 結構具名初始值

```c
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
    double score;
    int id;
} Student;

int main() {
    // ✅ 清楚知道每個欄位在設定什麼
    Student s = {
        .name = "小明",
        .score = 85.5,
        .age = 20,
        .id = 1001
    };
    // ❌ 傳統方式需要對照結構定義才知道順序
    // Student s = {"小明", 20, 85.5, 1001};

    printf("%s：%d 歲，%.1f 分 (ID: %d)\n",
           s.name, s.age, s.score, s.id);

    return 0;
}
```

> 💡 **最佳實踐**：總是使用具名初始值來初始化結構。這不僅可讀性更好，還能避免因結構成員順序調整而造成的錯誤。

### 巢狀具名初始值

```c
typedef struct { int x, y; } Point;
typedef struct {
    Point top_left;
    Point bottom_right;
    char label[32];
} Rectangle;

// 巢狀具名初始值
Rectangle r = {
    .top_left = {.x = 0, .y = 0},
    .bottom_right = {.x = 100, .y = 200},
    .label = "My Window"
};
```

## 彈性陣列成員（Flexible Array Member，C99）

結構的最後一個成員可以是「未指定長度」的陣列：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 傳統方式：固定大小，浪費空間或不夠用
#define MAX_NAME 32
typedef struct {
    int id;
    char name[MAX_NAME];  // 如果名字很短就浪費，太長又不夠
} FixedStudent;

// ✅ 彈性陣列成員：真正的可變長度字串
typedef struct {
    int id;
    int name_len;          // 可以記錄陣列長度
    char name[];           // ⭐ 彈性陣列成員——不佔 sizeof
} FlexStudent;

int main() {
    const char *student_name = "Alexander the Great";
    int len = strlen(student_name) + 1;

    // 動態配置：結構本體 + 彈性陣列的空間
    FlexStudent *s = malloc(sizeof(FlexStudent) + len);
    s->id = 1001;
    s->name_len = len;
    strcpy(s->name, student_name);

    printf("sizeof(FlexStudent) = %zu\n", sizeof(FlexStudent));
    // 只佔 8 bytes（id + name_len，不含 name 陣列）
    printf("學生：%s (ID: %d)\n", s->name, s->id);

    // 另一個學生可以用不同的長度
    FlexStudent *s2 = malloc(sizeof(FlexStudent) + 5);
    s2->id = 1002;
    strcpy(s2->name, "Bob");

    free(s);
    free(s2);
    return 0;
}
```

| 特性 | 說明 |
|------|------|
| 位置 | 必須是結構的**最後一個**成員 |
| 語法 | `char name[];` 或 `char name[0];`（GCC 舊語法） |
| `sizeof` | 不計入彈性陣列的大小 |
| 限制 | 包含彈性陣列的結構不能作為陣列元素或另一個結構的成員 |

## _Static_assert — 編譯期斷言（C11）

在編譯時期檢查條件，如果不符合就產生編譯錯誤：

```c
#include <stdio.h>
#include <assert.h>

// 整數型別大小檢查
_Static_assert(sizeof(int) == 4, "int 必須是 4 bytes！");
_Static_assert(sizeof(char) == 1, "char 必須是 1 byte！");

typedef struct {
    char name[20];
    int age;
    double score;
} Student;

// 確保結構沒有 padding（用於二進位讀寫）
_Static_assert(sizeof(Student) == 32, "Student 結構大小不如預期！");

int main() {
    printf("所有斷言通過，編譯成功！\n");
    return 0;
}
```

與執行時期 `assert()` 的比較：

```c
// 執行時期檢查
void divide(int a, int b) {
    assert(b != 0);        // 執行時期才檢查
    printf("%d\n", a / b);
}

// 編譯時期檢查（可以用在函式外部）
_Static_assert(sizeof(long) >= 4, "long 至少要 4 bytes");
```

> 💡 `_Static_assert` 的檢查是在編譯期，完全沒有執行時期開銷。適合用來檢查與平台相關的假設（如結構大小、型別大小、對齊方式）。

## _Generic — 泛型選擇（C11）

根據表達式的型別選擇不同的程式碼——C 語言版本的「多載」（overloading）：

```c
#include <stdio.h>
#include <math.h>

// 根據參數型別選擇不同的訊息
#define TYPE_INFO(x) _Generic((x), \
    int:    "整數", \
    double: "浮點數", \
    char*:  "字串", \
    float:  "單精度浮點數", \
    default: "其他型別" \
)

int main() {
    int a = 5;
    double b = 3.14;
    char *c = "Hello";
    float d = 2.5f;

    printf("a 是 %s\n", TYPE_INFO(a));  // 整數
    printf("b 是 %s\n", TYPE_INFO(b));  // 浮點數
    printf("c 是 %s\n", TYPE_INFO(c));  // 字串
    printf("d 是 %s\n", TYPE_INFO(d));  // 單精度浮點數

    return 0;
}
```

### 用 _Generic 實作多載函式

```c
#include <stdio.h>

// 一個 print_val 巨集，對不同型別有不同的輸出格式
#define print_val(x) _Generic((x), \
    int:    print_int, \
    double: print_double, \
    char*:  print_str    \
)(x)

void print_int(int x)    { printf("整數：%d\n", x); }
void print_double(double x) { printf("浮點數：%f\n", x); }
void print_str(char *x)  { printf("字串：%s\n", x); }

int main() {
    print_val(42);
    print_val(3.14);
    print_val("Hello, C11!");
    return 0;
}
```

### 實用範例：型別安全的 MAX 巨集

```c
#include <stdio.h>

#define MAX(x, y) _Generic((x) + (y), \
    int:    max_int, \
    double: max_double \
)(x, y)

int max_int(int a, int b) { return a > b ? a : b; }
double max_double(double a, double b) { return a > b ? a : b; }

int main() {
    printf("%d\n", MAX(3, 7));       // 7
    printf("%.1f\n", MAX(3.5, 2.1)); // 3.5
    return 0;
}
```

> 💡 `_Generic` 是 C 語言中為數不多的「編譯期多型」機制。搭配巨集使用，可以寫出類似 C++ 模板的效果，但型別選擇是在編譯期決定的。

## 匿名結構與匿名 Union（C11）

不用命名巢狀結構或 union 的成員名稱，直接存取內部成員：

```c
#include <stdio.h>

// 沒有匿名結構/union 時的寫法
typedef struct {
    union {
        int i;
        float f;
    } value;           // 需要中間名稱
    char type;
} OldNumber;

// ✅ 使用匿名 union（C11）
typedef struct {
    union {            // 沒有名稱！
        int i;
        float f;
    };                 // 不需要成員名稱
    char type;
} Number;

int main() {
    Number n;
    n.type = 'i';
    n.i = 42;          // ✅ 直接存取，不用 n.value.i
    printf("i = %d\n", n.i);

    n.type = 'f';
    n.f = 3.14f;       // ✅ 直接存取
    printf("f = %.2f\n", n.f);

    // 舊寫法對比
    OldNumber old;
    old.value.i = 42;  // ❌ 需要多一層 .value

    return 0;
}
```

### 實務應用：解析位元組

```c
#include <stdio.h>
#include <stdint.h>

typedef union {
    uint32_t value;
    struct {            // 匿名結構
        uint8_t byte0;
        uint8_t byte1;
        uint8_t byte2;
        uint8_t byte3;
    };                  // 不用命名這個結構
} Word;

int main() {
    Word w;
    w.value = 0x12345678;

    // 直接存取每個位元組
    printf("byte0: 0x%02x\n", w.byte0);  // 0x78（小端序）
    printf("byte1: 0x%02x\n", w.byte1);  // 0x56
    printf("byte2: 0x%02x\n", w.byte2);  // 0x34
    printf("byte3: 0x%02x\n", w.byte3);  // 0x12

    return 0;
}
```

## inline 函式（C99）

提示編譯器將函式內容直接展開到呼叫處，避免函式呼叫開銷：

```c
// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

inline int square(int x) {
    return x * x;
}

// 巨集的替代方案
inline int max(int a, int b) {
    return a > b ? a : b;
}

#endif
```

| 比較 | 巨集 | `inline` 函式 |
|------|------|--------------|
| 型別檢查 | ❌ 無 | ✅ 有 |
| 副作用問題 | ❌ 參數可能多次求值 | ✅ 只求值一次 |
| 偵錯 | ❌ 展開後很難追蹤 | ✅ 可設中斷點 |
| 適用情境 | 需要泛型時（搭配 `_Generic`） | 一般小型函式 |

> ⚠️ `inline` 只是對編譯器的**建議**，不是強制。現代編譯器會自行判斷是否內聯。另外，`inline` 函式的定義通常放在 `.h` 檔中。

## alignas / alignof — 對齊控制（C11）

```c
#include <stdio.h>
#include <stdalign.h>

// 指定結構的對齊方式
struct alignas(64) CacheLine {
    int data[16];
};

int main() {
    printf("CacheLine 對齊：%zu\n", alignof(struct CacheLine));  // 64

    // 檢查型別的對齊需求
    printf("int 對齊：%zu\n", alignof(int));        // 通常 4
    printf("double 對齊：%zu\n", alignof(double));  // 通常 8

    return 0;
}
```

## _Noreturn — 不返回的函式（C11）

```c
#include <stdio.h>
#include <stdlib.h>

_Noreturn void fatal_error(const char *msg) {
    fprintf(stderr, "致命錯誤：%s\n", msg);
    exit(1);
    // 這個函式不會 return！
}

int main() {
    int *p = malloc(999999999999L);
    if (p == NULL) {
        fatal_error("記憶體不足");
    }
    // 編譯器知道 fatal_error 不會返回，不會警告「缺少 return」
    return 0;
}
```

> 💡 C23 之後推薦使用 `[[noreturn]]` 屬性語法代替 `_Noreturn`。

## C23 語法預覽

C23 是 2024 年發布的最新標準，加入了一些現代化語法：

```c
// 1. nullptr（取代 NULL）
#include <stddef.h>
int *p = nullptr;          // C23 起可用
// 與 NULL 不同，nullptr 的型別是 nullptr_t，而不是 int

// 2. constexpr（編譯期常數）
constexpr int ARRAY_SIZE = 100;
int arr[ARRAY_SIZE];       // C23 之前需要 enum 或 #define

// 3. 屬性語法（類似 C++）
[[deprecated("改用 new_func()")]]
void old_func() {}

[[nodiscard]] int get_status() {
    return 1;
}

// 4. #elifdef / #elifndef（C23）
#ifdef DEBUG
    #define LOG(x) printf("%s\n", x)
#elifdef RELEASE
    #define LOG(x) // nothing
#endif

// 5. auto 從 C++ 借來的型別推導（僅限檔案層級）
// auto x = 42;  // C23 允許，但僅限於宣告時就能推導型別的情境
```

## 實務應用：指令解碼器

結合 `_Generic`、匿名 union、`_Static_assert` 的實際範例：

```c
#include <stdio.h>
#include <stdint.h>
#include <assert.h>

// 不同類型的指令封包
typedef struct { uint8_t opcode; uint32_t addr; } MemInstruction;
typedef struct { uint8_t opcode; uint8_t reg1; uint8_t reg2; } RegInstruction;
typedef struct { uint8_t opcode; int32_t offset; } BranchInstruction;

// 共用體——解析傳入的指令
typedef union {
    uint8_t raw[8];          // 原始位元組
    struct {                 // 匿名結構
        uint8_t opcode;
        uint8_t payload[7];
    };
    MemInstruction mem;
    RegInstruction reg;
    BranchInstruction br;
} Instruction;

// C23 可以用 static_assert（C11 用 _Static_assert）
_Static_assert(sizeof(Instruction) == 8, "Instruction 必須是 8 bytes");

// 用 _Generic 選擇解碼方式（示意）
#define decode(inst) _Generic((inst).opcode, \
    default: printf("未知指令：0x%02x\n", (inst).opcode) \
)

int main() {
    Instruction inst = {.raw = {0x01, 0x10, 0x20, 0x00, 0x00, 0x00, 0x00, 0x00}};

    printf("opcode = 0x%02x\n", inst.opcode);
    printf("payload[0] = 0x%02x\n", inst.payload[0]);

    decode(inst);

    return 0;
}
```

## 練習題

### 練習 1：複合字面常數

不使用暫存變數，在單一 `printf` 對一個匿名 `Point` 結構（`int x, y`）的兩個成員進行加總並輸出。

### 練習 2：具名初始值

宣告一個 `Book` 結構（書名、作者、年份、ISBN），使用具名初始值建立一本書的資料。

### 練習 3：彈性陣列成員

設計一個 `Buffer` 結構，內部含有一個大小欄位和一個彈性陣列成員，提供 `create_buffer(size)` 和 `free_buffer(buf)` 兩個函式。

### 練習 4：_Generic

使用 `_Generic` 實作一個 `to_string(x)` 巨集，對 `int`、`double`、`char*` 分別輸出不同格式的字串。
